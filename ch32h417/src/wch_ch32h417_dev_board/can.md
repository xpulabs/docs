# CH32H417 CAN控制器实操教程

本教程基于CH32H417系列MCU，详细讲解CAN控制器的核心原理、工作模式、实操配置及常见问题排查，适用于工业、汽车等领域的CAN通信开发，帮助开发者快速掌握CAN控制器的使用方法，实现稳定的串行数据通信。

# 第一章：CAN控制器基础认知

## 1.1 什么是CAN控制器

控制器局域网（CAN）是一种高性能、高可靠性的串行数据通信协议，CH32H417内置3组CAN控制器，完整实现CAN协议2.0A（标准帧）和2.0B（扩展帧，主动模式），可构建强大的分布式实时控制局域网，以较低的CPU负荷处理大量数据报文，广泛应用于工业控制、汽车电子等场景。

## 1.2 核心特性（必记）

- 硬件资源：3组独立CAN控制器，支持CAN 2.0A/B协议，最高传输速率可达1Mbit/s。

- 发送能力：每组CAN支持3个发送邮箱，可设置发送优先级，记录发送报文SOF时刻时间戳。

- 接收能力：每组CAN支持2个3级邮箱深度的接收FIFO，共42组过滤器，支持32/16位过滤模式，减少软件筛选压力。

- 中断支持：4个独立中断向量（发送、FIFO0、FIFO1、错误及状态变化），各中断源可独立配置。

- 特殊功能：支持时间触发通信（避免低优先级消息阻塞）、低功耗睡眠模式、多种测试模式，适配不同应用场景。

# 第二章：CAN控制器工作模式详解

CH32H417的CAN控制器通过操作CANx_CTLR寄存器的SLEEP和INRQ位，可在*初始化模式、睡眠模式、正常模式*三者间切换，模式切换是CAN配置的基础，需严格遵循操作流程。

## 2.1 初始化模式（配置核心）

初始化模式用于配置CAN的核心参数（如位时序、过滤器、邮箱等），此模式下禁止报文收发。

- 进入方式：对CANx_CTLR寄存器的INRQ位置1，查询CANx_STATR寄存器的INAK位，当INAK自动置1时，成功进入初始化模式。

- 退出方式：对CANx_CTLR寄存器的INRQ位清0，查询CANx_STATR的INAK位，当INAK自动清0时，退出初始化模式，进入正常模式。

- 注意事项：过滤器组初始化可在非初始化模式下进行，但需将CANx_FCTLR寄存器的FINIT位置1，此时禁止接收报文。

## 2.2 睡眠模式（低功耗）

睡眠模式用于降低功耗，此时CAN控制器时钟停止，但邮箱寄存器仍可访问，TX引脚输出隐性位。

- 进入方式：对CANx_CTLR寄存器的SLEEP位置1，查询CANx_STATR的SNAK位，SNAK自动置1则进入睡眠模式。

- 退出方式：

    - 进入初始化模式：SLEEP位清0，INRQ位置1，查询INAK位置1即可。

    - 进入正常模式：SLEEP位清0，查询SNAK位清0即可。

## 2.3 正常模式（通信核心）

正常模式下，CAN控制器可正常收发报文，完成数据通信。进入正常模式的前提是：退出初始化模式且未进入睡眠模式（SLEEP=0、INRQ=0，SNAK=0、INAK=0）。

# 第三章：测试模式（调试必备）

测试模式用于CAN控制器的调试和总线状态分析，需在初始化模式下配置，配置完成后退出初始化模式即可进入测试模式，共3种测试模式。

## 3.1 静默模式（总线状态分析）

- 配置方式：初始化模式下，将CANx_BTIMR寄存器的SILM位置1，退出初始化模式即进入。

- 核心特点：可接收报文，不能对外发送报文（TX引脚始终输出隐性位），不影响总线，适合分析总线状态。

## 3.2 环回模式（收发测试）

- 配置方式：初始化模式下，将CANx_BTIMR寄存器的LBKM位置1，退出初始化模式即进入。

- 核心特点：可对外发送报文，不能接收外部报文，但自身发送的报文可被本节点接收，接收过滤机制有效，适合测试CAN收发功能。

## 3.3 静默环回模式（封闭自测试）

- 配置方式：初始化模式下，将CANx_BTIMR寄存器的SILM和LBKM位同时置1，退出初始化模式即进入。

- 核心特点：对CAN总线无影响，RX引脚与总线断开，TX引脚置隐性位，适合CAN控制器的封闭自测试，无需外部总线。

# 第四章：核心功能实操配置（重点）

CAN通信的核心流程为：**初始化配置→发送报文→接收报文**，以下结合寄存器配置，讲解关键步骤（以CAN1为例）。

## 4.1 初始化配置（必做步骤）

初始化配置需在初始化模式下完成，核心步骤包括：时钟使能→进入初始化模式→位时序配置→过滤器配置→退出初始化模式。

### 4.1.1 步骤1：使能CAN时钟

CAN控制器的时钟由APB1提供，需先使能APB1时钟和对应CAN控制器时钟（如CAN1），同时若使用过滤器，需使能CAN1时钟（过滤器时钟由CAN1提供）。

示例配置（寄存器操作）：

- 使能APB1时钟：RCC->APB1ENR |= (1<<25); // CAN1时钟使能

- 使能GPIO时钟（CAN引脚复用）：根据CAN引脚（如PA11=CAN1_RX，PA12=CAN1_TX），使能对应GPIO端口时钟，并配置为复用功能。

### 4.1.2 步骤2：进入初始化模式

操作CAN1_CTLR寄存器，请求进入初始化模式，并等待确认：

```c
CAN1->CTLR |= (1<<0); // INRQ位置1，请求进入初始化模式
while( (CAN1->STATR & (1<<0)) == 0 ); // 等待INAK位置1，确认进入初始化模式
```

### 4.1.3 步骤3：位时序配置（关键，决定通信速率）

CAN总线每一位时间分为3段：同步段（SS，固定1Tq）、时间段1（BS1）、时间段2（BS2），Tq为最小时间单元（Tq=(BRP+1)×tHCLK），需通过CANx_BTIMR寄存器配置。

- BRP[9:0]：配置Tq，BRP值=（APB1时钟频率/(波特率×(1+BS1+BS2))）-1。

- TS1[3:0]：BS1长度=TS1+1（范围1~16Tq）。

- TS2[3:0]：BS2长度=TS2+1（范围1~8Tq）。

- SJW[3:0]：重新同步跳转宽度=SJW+1（范围1~4Tq）。

示例：APB1时钟为42MHz，波特率为500Kbit/s，配置BS1=13Tq，BS2=2Tq，SJW=1Tq：

```c
CAN1->BTIMR = 0;
CAN1->BTIMR |= (0<<24); // SJW=1Tq（SJW[3:0]=0）
CAN1->BTIMR |= (12<<16); // BS1=13Tq（TS1[3:0]=12）
CAN1->BTIMR |= (1<<20); // BS2=2Tq（TS2[3:0]=1）
CAN1->BTIMR |= (41<<0); // BRP=41，Tq=(41+1)×(1/42MHz)=1us
```

### 4.1.4 步骤4：过滤器配置（接收报文筛选）

CH32H417共42组过滤器，每组可配置为32/16位模式、屏蔽位/标识符列表模式，需通过CAN1_FCTLR、CAN1_FMCFGR、CAN1_FSCFGR等寄存器配置（过滤器仅CAN1可配置，分配给3组CAN使用）。

示例：配置过滤器组0为32位屏蔽位模式，关联到FIFO0，过滤标准标识符0x123：

```c
// 1. 进入过滤器初始化模式
CAN1->FCTLR |= (1<<0); // FINIT位置1
// 2. 配置过滤器0为32位屏蔽位模式
CAN1->FMCFGR &= ~(1<<0); // FBM0=0，屏蔽位模式
CAN1->FSCFGR |= (1<<0); // FSC0=1，32位模式
// 3. 关联到FIFO0
CAN1->FAFIFOR &= ~(1<<0); // FFA0=0，关联FIFO0
// 4. 配置标识符和屏蔽字（标准帧，IDE=0）
CAN1->F0R1 = 0x123<<21; // 标识符寄存器：标准帧0x123
CAN1->F0R2 = 0x1FFFFFFF; // 屏蔽字：全1，严格匹配标识符
// 5. 激活过滤器0
CAN1->FWR |= (1<<0); // FACT0=1，激活过滤器
// 6. 退出过滤器初始化模式
CAN1->FCTLR &= ~(1<<0); // FINIT位清0
```

### 4.1.5 步骤5：退出初始化模式，进入正常模式

```c
CAN1->CTLR &= ~(1<<0); // INRQ位清0，请求退出初始化模式
while( (CAN1->STATR & (1<<0)) != 0 ); // 等待INAK位清0，确认进入正常模式
```

## 4.2 发送报文配置

每组CAN有3个发送邮箱（y=0/1/2），发送流程：选择空邮箱→配置报文参数→请求发送→等待发送完成。

```c
// 1. 等待发送邮箱为空（以邮箱0为例）
while( (CAN1->TSTATR & (1<<26)) == 0 ); // 等待TME0位为1，邮箱0为空
// 2. 配置报文标识符（标准帧，IDE=0，RTR=0（数据帧））
CAN1->TXMI0R = 0x123<<21; // 标识符0x123，TXRQ=0（暂不请求发送）
// 3. 配置数据长度（DLC=4，发送4字节数据）
CAN1->TXMDT0R = 4<<0; // DLC[3:0]=4
// 4. 写入发送数据（低4字节：DATA0~DATA3）
CAN1->TXMDL0R = 0x12345678; // DATA0=0x78，DATA1=0x56，DATA2=0x34，DATA3=0x12
// 5. 请求发送（TXRQ位置1）
CAN1->TXMI0R |= (1<<0);
// 6. 等待发送完成（可选）
while( (CAN1->TSTATR & (1<<16)) == 0 ); // 等待RQCP0位为1，发送完成
CAN1->TSTATR |= (1<<16); // 清除发送完成标志
```

## 4.3 接收报文配置

接收报文由硬件自动完成，存储在接收FIFO中，软件通过读取FIFO邮箱获取报文，流程：等待FIFO有报文→读取报文参数→释放邮箱。

```c
// 1. 等待FIFO0有报文（FMP0[1:0] != 0）
while( (CAN1->RFIFO0 & (3<<0)) == 0 ); // 等待FMP0不为0，FIFO0有报文
// 2. 读取报文标识符
uint32_t id = CAN1->RXMI0R >> 21; // 标准帧标识符（IDE=0）
// 3. 读取数据长度
uint8_t dlc = CAN1->RXMDT0R & 0x0F;
// 4. 读取接收数据
uint32_t data_low = CAN1->RXMDL0R; // DATA0~DATA3
uint32_t data_high = CAN1->RXMDH0R; // DATA4~DATA7
// 5. 释放接收邮箱
CAN1->RFIFO0 |= (1<<5); // RFOM0位置1，释放FIFO0当前邮箱
```

## 4.4 中断配置（可选，高效处理）

CAN支持4个中断向量，可配置发送中断、FIFO接收中断、错误中断等，以FIFO0接收中断为例：

```c
// 1. 使能FIFO0接收中断（FMPIE0位）
CAN1->INTENR |= (1<<1); // FMPIE0=1，FIFO0有报文时产生中断
// 2. 配置NVIC中断优先级（CAN1_RX0_IRQn）
NVIC_InitTypeDef NVIC_InitStruct;
NVIC_InitStruct.NVIC_IRQChannel = CAN1_RX0_IRQn;
NVIC_InitStruct.NVIC_IRQChannelPreemptionPriority = 1;
NVIC_InitStruct.NVIC_IRQChannelSubPriority = 0;
NVIC_InitStruct.NVIC_IRQChannelCmd = ENABLE;
NVIC_Init(&NVIC_InitStruct);
// 3. 中断服务函数
void CAN1_RX0_IRQHandler(void)
{
    // 读取报文（同接收配置步骤）
    uint32_t id = CAN1->RXMI0R >> 21;
    uint8_t dlc = CAN1->RXMDT0R & 0x0F;
    uint32_t data = CAN1->RXMDL0R;
    // 释放邮箱
    CAN1->RFIFO0 |= (1<<5);
}
```

# 第五章：进阶功能解析

## 5.1 发送优先级配置

发送优先级由CANx_CTLR寄存器的TXFP位决定：

- TXFP=1：按发送请求先后次序发送，适合分段发送。

- TXFP=0：按报文标识符优先级发送（标识符越小，优先级越高），同标识符时，邮箱编号越小优先级越高。

## 5.2 时间触发模式（避免阻塞）

时间触发模式用于解决总线繁忙时低优先级消息阻塞问题，需配合TTCAN协议使用，配置步骤：

1. 使能时间触发模式：CANx_CTLR寄存器的TTCM位置1，NART位置1（禁止自动重传）。

2. 选择模式：CANx_TTCTLR寄存器的MODE位，0=默认模式，1=增强模式。

3. 配置参数：默认模式下，内部定时器自动产生时间戳；增强模式下，需配置时间基准节点，同步各节点计数器，设置触发计数值（CANx_TTCNT）和计数终值（CANx_TTCTLR）。

## 5.3 错误处理机制

CAN控制器通过CANx_ERRSR寄存器（TEC发送错误计数器、REC接收错误计数器）管理总线错误，错误状态分为3种：

- 错误主动状态（TEC/REC < 128）：正常通信，检测到错误时发送主动错误标志。

- 错误被动状态（TEC/REC > 127）：可通信，检测到错误时发送被动错误标志。

- 离线状态（TEC > 255）：无法通信，需手动或自动退出（ABOM位控制）。

注：当总线检测到128次连续11个隐性位时，可恢复为错误主动状态。

# 第六章：常见问题排查（快速定位）

1. **无法发送报文**

    - 排查：未退出初始化模式（INAK=1）；发送邮箱未空（TMEy=0）；TXRQ位未置1；位时序配置错误（波特率不匹配）。

    - 解决：确认退出初始化模式；等待邮箱为空再发送；检查TXRQ位配置；重新计算位时序参数。

2. **无法接收报文**

    - 排查：过滤器未配置或未激活；过滤器模式/位宽配置错误；FIFO未释放（RFOM位未置1）；CAN引脚复用配置错误。

    - 解决：重新配置过滤器并激活；检查过滤器关联FIFO是否正确；读取报文后释放邮箱；确认GPIO复用为CAN引脚。

3. **报文丢失/溢出**

    - 排查：接收FIFO满未及时释放（FULLx=1）；FIFO溢出（FOVRx=1）；RFLM位配置错误（锁定模式未启用）。

    - 解决：及时读取并释放FIFO邮箱；启用FIFO锁定模式（RFLM=1），避免报文覆盖；优化中断响应速度。

4. **总线错误（TEC/REC异常）**

    - 排查：总线终端电阻未接（需120Ω）；位时序不匹配；总线存在干扰；CAN引脚接线错误（RX/TX接反）。

    - 解决：添加终端电阻；重新配置位时序；优化布线，远离干扰源；检查RX/TX接线。

# 第七章：核心寄存器速查（实操常用）

## 7.1 控制类寄存器

|寄存器名称|访问地址（CAN1）|核心功能|
|---|---|---|
|CANx_CTLR|0x40006400|主控制寄存器：模式切换（INRQ/SLEEP）、时间触发（TTCM）、自动重传（NART）等。|
|CANx_STATR|0x40006404|主状态寄存器：模式状态（INAK/SNAK）、引脚电平（RX/SAMP）、中断标志（SLAKI/WKUI）等。|
|CANx_BTIMR|0x4000641C|位时序寄存器：配置波特率（BRP）、BS1、BS2、SJW，测试模式（SILM/LBKM）。|
## 7.2 收发类寄存器

|寄存器名称|访问地址（CAN1）|核心功能|
|---|---|---|
|CANx_TSTATR|0x40006408|发送状态寄存器：邮箱空（TMEy）、发送完成（RQCPy）、错误标志（TERRy/ALSTy）等。|
|CANx_RFIFO0/1|0x4000640C/0x40006410|接收FIFO状态寄存器：FIFO报文数（FMPx）、满标志（FULLx）、溢出标志（FOVRx）、释放邮箱（RFOMx）。|
|CANx_TXMIyR|0x40006580（y=0）|发送邮箱标识符寄存器：配置标识符、IDE、RTR、发送请求（TXRQ）。|
|CANx_RXMIyR|0x400065B0（y=0）|接收邮箱标识符寄存器：读取标识符、IDE、RTR。|
## 7.3 过滤器类寄存器（CAN1独有）

|寄存器名称|访问地址|核心功能|
|---|---|---|
|CAN1_FCTLR|0x40006600|过滤器主控制寄存器：过滤器初始化（FINIT）、过滤器组分配（CAN2SB/CAN3SB）。|
|CAN1_FMCFGR|0x40006604|过滤器模式寄存器：配置过滤器组的模式（屏蔽位/标识符列表）。|
|CAN1_FWR|0x4000661C|过滤器激活寄存器：激活/禁用各过滤器组（FACTx）。|
# 总结

CH32H417 CAN控制器的核心是“硬件实现CAN协议，软件配置参数”，实操关键在于：掌握模式切换流程、正确配置位时序（保证通信速率匹配）、合理设置过滤器（减少软件负担）、及时处理收发中断和错误。

本教程覆盖了CAN通信的基础配置、核心功能和常见问题，开发者可根据实际需求（如时间触发、低功耗、多节点通信），结合芯片手册扩展配置，实现稳定、高效的CAN通信。
