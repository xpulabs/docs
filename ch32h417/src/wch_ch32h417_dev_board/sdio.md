# CH32H417/CH32H415 SDIO接口新手教程

本教程专门针对CH32H417、CH32H415微控制器的SDIO接口，从基础认知、核心特性、协议原理、实操流程到寄存器配置、常见问题，全面讲解SDIO的使用方法，适用于新手快速入门，可直接用于SD卡、TF卡、eMMC卡读写及SDIO接口设备（如WiFi/4G模块）控制开发。

注：本教程所述“SDIO”均指CH32H417/CH32H415微控制器内置的SDIO外设，该外设仅适用于这两款型号，且仅作为SDIO主机使用，被控制的设备（SD卡、SDIO模块等）统称SDIO设备。

# 第一章：SDIO接口基础认知

## 1.1 什么是SDIO接口

SDIO（Secure Digital Input/Output）是微控制器上专门用于操作外部存储卡（SD卡、TF卡、eMMC卡）或SDIO接口设备（WiFi/4G模块）的通信接口，属于微控制器的核心外设之一。

CH32H417/CH32H415的SDIO接口核心特点：

- 总线挂载：直接挂载在HB总线上，由HCLK直接提供时钟，通信速度高；

- 主机角色：仅作为SDIO主机，所有传输均由微控制器主动发起；

- 核心用途：读写SD/TF/eMMC卡、控制SDIO接口的WiFi/4G等非存储设备；

- 与QSPI区别：无片选引脚，新增CMD引脚（专门传输命令/响应），支持1/4/8位总线宽度。

## 1.2 核心特征

### 1.2.1 关键特性

- 设备兼容：支持SD卡、SDIO卡和MMC卡，兼容MMC规范4.5（向前兼容）、SD卡规范2.0、SDIO卡规范2.0；

- 总线宽度：支持1位、4位和8位总线，可根据传输速度需求切换；

- 时钟速度：最高通讯时钟可达100MHz，实际使用中建议限制在50MHz以内（匹配IO输出能力）；

- 模式限制：不兼容SPI或QSPI模式，需与SPI接口区分使用；

- FIFO配置：内置32个32位FIFO，支持CPU或DMA读写，提升数据传输效率。

### 1.2.2 核心概述

SDIO接口仅提供SD/MMC规范所需的时钟、数据和命令控制时序，各命令间的先后组合需通过程序自行定义；仅实现存储卡的底层读写功能，文件系统（如FAT32）需用户自行通过程序构建。

时钟使用原则：初始化阶段时钟需低于400kHz，正式数据传输时可提升至设备支持的最大时钟（最大为HCLK的一半）；时钟超过阈值时，需降低时钟线、数据线、命令线的波形峰值，减少上升/下降耗时。

设备优先级：本教程叙述优先针对SD卡，其次为SDIO卡（非存储设备），最后为MMC卡；未特殊说明时，内容均适用于SD卡。

## 1.3 接口结构与引脚配置

### 1.3.1 外设结构

SDIO外设由5个核心模块组成，半双工通信，数据流通过通用DMA完成RAM与SDIO FIFO的搬移：

- HB接口：接收CPU对SDIO的控制；

- 时钟控制部分：生成SDIO_CK时钟信号，由HCLK分频得到；

- CMD线控制部分：处理CMD引脚的命令发送和响应接收，附加CRC校验；

- 数据线控制部分：处理SDIO_D[7:0]引脚的数据传输，附加CRC校验；

- 控制寄存器部分：存储配置参数、状态信息，实现中断、DMA等功能控制。

### 1.3.2 引脚配置

SDIO需配置的引脚及模式如下，共10个引脚（SDIO_CK、SDIO_CMD、SDIO_D[7:0]）：

|SDIO复用功能|引脚模式|
|---|---|
|SDIO_D[0:3]|推挽复用输出|
|SDIO_CK|推挽复用输出|
|SDIO_CMD|推挽复用输出|
|SDIO_D4~SDIO_D7|推挽复用输出|
注：若工作在从模式（极少用），SDIO_CK引脚需配置为浮空输入。

## 1.4 时钟配置

SDIO的时钟（SDIO_CK）由HCLK分频得到，分频系数可配置为2~261之间的任意整数，计算公式为：SDIO_CK = HCLK /（CLKDIV + 2），其中CLKDIV为时钟寄存器（R32_SDIO_CLKCR）的分频域。

时钟配置注意事项：

- 初始化阶段：SDIO设备仅支持最高400kHz时钟，需配置合适的分频系数；

- 传输阶段：电压切换完成后，可将时钟提升至设备支持的最大速度（如SDHC卡UHS-I模式可到80MHz，CH32建议不超过50MHz）；

- 时钟修改：读写数据期间及之后7个HCLK周期内，不可修改时钟寄存器配置；

- 节能配置：可通过PWRSAV位配置，总线空闲时关闭SDIO_CK输出，节省电能。

# 第二章：SDIO协议核心原理

SDIO通信以“传输”为最小单位，每个传输均以主机在CMD线上发送命令为起始，部分命令会伴随设备的响应（CMD线传输）和数据传输（D线传输）；命令、响应、数据均有固定格式，超时会产生错误。

注：CH32H417/CH32H417仅实现SD 2.0、SDIO 2.0、MMC 4.5规范的硬件基础，不支持双边沿采样等更高版本规范功能，开发时需以对应规范为依据。

## 2.1 命令（CMD）

### 2.1.1 命令格式

命令总长48位，固定格式如下（高位在前，低位在后）：

|位/域名称|位次/宽度|固定值|说明|
|---|---|---|---|
|起始位|47（1位）|0b|命令起始标志|
|传输位|46（1位）|1b|表示主机向设备传输|
|命令索引|45:40（6位）|X|区分不同命令（如CMD0、CMD17）|
|命令参数|39:8（32位）|X|命令对应的具体参数（如地址、长度）|
|CRC7校验|7:1（7位）|X|对命令索引和参数的CRC校验|
|结束位|0（1位）|1b|命令结束标志|
### 2.1.2 命令类型

命令分为4类，核心用途如下：

- 广播命令（bc）：发给总线上所有设备，无响应返回（如CMD0，复位所有卡到IDLE状态）；

- 带响应的广播命令（bcr）：发给所有设备，有响应返回（如CMD2，所有卡回复CID）；

- 点对点命令（ac）：发给特定设备，无数据传输（如CMD7，选中/取消选中某张卡）；

- 带数据传输的点对点命令（adtc）：发给特定设备，伴随数据传输（如CMD17，单块读）。

### 2.1.3 常用命令（SD卡重点）

开发中最常用的命令分类整理，参数和响应类型需严格匹配：

#### 1. 基础命令（初始化必用）

|命令索引|类型|响应类型|核心功能|
|---|---|---|---|
|CMD0|bc|无|复位所有卡到IDLE状态（初始化第一步）|
|CMD2|bcr|R2|所有卡回复CID（卡身份识别寄存器）|
|CMD3|bcr|R6|卡回复新的RCA（相对卡地址，用于选中特定卡）|
|CMD7|ac|R1b|选中或取消选中特定卡（需传入RCA）|
|CMD8|bcr|R7|发送SD卡接口条件，检测卡支持的电压|
|CMD11|ac|R1|切换SD卡接口电平到1.8V（提升时钟前提）|
#### 2. 块传输命令（读写必用）

|命令索引|类型|响应类型|核心功能|
|---|---|---|---|
|CMD16|ac|R1|设置块长度（常用512字节，SDHC/SDXC必设）|
|CMD17|adtc|R1|单块读（传入块地址，读取1个数据块）|
|CMD18|adtc|R1|多块读（传入起始地址，连续读取多个块）|
|CMD24|adtc|R1|单块写（传入块地址，写入1个数据块）|
|CMD25|adtc|R1|多块写（传入起始地址，连续写入多个块）|
|CMD12|ac|R1b|强制停止数据传输（多块读写后必用）|
#### 3. 擦除命令（大批量写前优化）

SD卡为FLASH结构，写入前会自动擦除，但主动擦除可提升大批量写入效率，仅适用于SD卡：

- CMD32：设定擦除首地址（响应R1）；

- CMD33：设定擦除尾地址（响应R1）；

- CMD38：执行擦除操作（响应R1b，参数为0时为普通擦除）。

注：SDHC/SDXC卡（2GB~2TB）的擦除地址必须是512字节对齐的块地址。

## 2.2 响应（Response）

响应是SDIO设备对主机命令的回复，仅在CMD线上传输，需在命令结束后规定时间内发出，否则触发超时错误；所有响应均以0b为起始位、0b为传输位（设备向主机传输）、1b为结束位，高位在前。

SD卡支持R1/R1b/R2/R3/R6/R7共6种响应，SDIO卡额外支持R4/R5，核心响应类型详解如下（实操重点）：

### 2.2.1 R1响应（最常用）

普通响应，总长48位，带CRC7校验，卡状态域32位，对应大部分基础命令和块传输命令（如CMD16、CMD17）：

|位/域名称|位次/宽度|说明|
|---|---|---|
|起始位|47（1位）|固定0b|
|传输位|46（1位）|固定0b（设备→主机）|
|响应索引|45:40（6位）|跟随对应命令的索引（如CMD17的响应索引为17）|
|卡状态|39:8（32位）|SD卡当前状态（如空闲、忙、错误）|
|CRC7校验|7:1（7位）|对响应索引和卡状态的校验|
|结束位|0（1位）|固定1b|
### 2.2.2 其他常用响应

- R1b响应：与R1格式一致，区别是响应后会添加繁忙信号（D2引脚低电平），主机需等待繁忙结束（如CMD7、CMD38）；

- R2响应：总长136位，无独立CRC7校验（包含在卡状态域），用于回复CID/CSD寄存器（如CMD2回复CID、CMD9回复CSD）；

- R3响应：总长48位，无CRC7校验，专门回复OCR寄存器（如ACMD41，用于检测卡上电状态和支持电压）；

- R6响应：总长48位，带CRC7校验，专门回复RCA（如CMD3，返回卡的相对地址）；

- R7响应：总长48位，带CRC7校验，专门回复CMD8，表明卡支持的电压范围。

## 2.3 数据传输

数据传输在SDIO_D[7:0]数据线上进行，支持1/4/8位总线宽度，SD卡仅支持块传输模式（带CRC校验），MMC卡额外支持数据流传输模式（无CRC）。

### 2.3.1 传输规则

- 数据顺序：每个时钟周期先传输起始位，每个字节高位在前、低位在后；

- CRC校验：SD卡块传输结束后，会附加CRC校验位，主机需验证CRC正确性；

- 任意长度传输：开启RANDOM_LEN_EN位，可配置任意字节长度（0~2048字节）的块传输，长度由DBLOCKSIZE2定义；

- 传输配置：需设置传输总长度（R32_SDIO_DLEN）、块大小，使能DTEN位，向FIFO写入/读取数据。

### 2.3.2 传输时序（核心）

实操中重点关注以下3种时序，确保命令、响应、数据的时序匹配：

- 单块读：主机发CMD17（带块地址）→ 卡回复R1 → 卡发送数据块 → 卡发送CRC → 传输结束；

- 单块写：主机发CMD24（带块地址）→ 卡回复R1 → 主机发送数据块 → 主机发送CRC → 卡回复确认；

- 多块读写：主机发CMD18/CMD25 → 卡回复R1 → 连续传输多个数据块（每个块带CRC）→ 主机发CMD12停止传输 → 卡回复R1b。

## 2.4 状态机（了解即可）

SDIO的命令和数据传输遵循固定状态机，无需手动控制状态切换，只需配置正确的寄存器，硬件会自动完成状态流转：

- 命令状态机（CPSM）：控制命令发送、响应接收，核心状态包括Idle（空闲）、Send（发送命令）、Wait response（等待响应）等；

- 数据状态机（DPSM）：控制数据传输，核心状态包括Idle（空闲）、Read wait（读等待）、Send（发送数据）、Receive（接收数据）等。

## 2.5 从模式（极少用）

开启SLV_MODE位（数据控制寄存器2），SDIO外设进入从模式，等待外部主机命令：

- 接收命令：接收到的命令参数存入RESP1，命令索引存入RESP2；

- 响应方式：自动向外部主机回复R1类型响应，响应参数为CMDARG的值；

- 错误控制：可通过SLV_FORCE_ERR位强制数据块CRC错误，拒绝不期望的读写操作。

注：从模式仅支持R1响应，主从通信的命令索引和参数含义需软件自行定义。

# 第三章：SDIO设备初始化与核心寄存器配置

SDIO开发的核心是“寄存器配置+命令交互”，先熟悉核心寄存器的功能，再掌握初始化流程，即可完成基础的读写操作。CH32H417/CH32H415的SDIO每个模块包含16个核心寄存器，前缀为R32_SDIO_，重点讲解实操必用寄存器。

## 3.1 核心寄存器详解（实操必用）

### 3.1.1 电源寄存器（R32_SDIO_POWER）

地址：0x40018000，复位值：0x00000000，核心配置位（仅列实操常用）：

|位字段|功能描述|实操配置要点|
|---|---|---|
|PWRCTRL[1:0]（bit1~0）|电源检测与时钟控制：00=电源关闭；11=上电，卡时钟开启|初始化时置11b，开启SDIO电源和时钟；禁用时置00b|
### 3.1.2 时钟寄存器（R32_SDIO_CLKCR）

地址：0x40018004，复位值：0x00000000，核心配置位（时钟控制核心）：

|位字段|功能描述|实操配置要点|
|---|---|---|
|CLKDIV[7:0]（bit7~0）|时钟分频系数，SDIO_CK = HCLK/(CLKDIV+2)|初始化时设为0x9F（HCLK=144MHz时，400kHz）；传输时设为0x0E（50MHz）|
|CLKEN（bit8）|时钟使能：1=允许SDIO_CK输出；0=禁止|配置完分频后置1，禁用时置0|
|WIDBUS[1:0]（bit11~10）|总线宽度：00=1位；01=4位；10=8位|初始化用1位（00b），传输时可切换为4位/8位提升速度|
|PWRSAV（bit9）|空闲时钟节能：1=空闲时关闭SDIO_CK；0=始终输出|低功耗场景置1，追求稳定性置0|
### 3.1.3 命令参数寄存器（R32_SDIO_ARG）

地址：0x40018008，复位值：0x00000000，核心功能：

- 主机模式：存储命令的32位参数（如块地址、RCA、块长度），发送命令时自动随命令一起传输；

- 从机模式：存储从机回复应答的32位响应参数。

实操要点：发送命令前，需先将对应参数写入该寄存器（如发送CMD17时，写入块地址）。

### 3.1.4 命令寄存器（R32_SDIO_CMD）

地址：0x4001800C，复位值：0x00000000，核心配置位（命令发送控制）：

|位字段|功能描述|实操配置要点|
|---|---|---|
|CMDINDEX[5:0]（bit5~0）|命令索引，填写要发送的命令编号（如CMD0=0x00）|发送命令前，按需求填写对应命令索引|
|WAITRESP[1:0]（bit7~6）|响应类型：00=无响应；01=短响应；11=长响应||
