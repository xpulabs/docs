# CH32H417系列SAI教程（从入门到实操）

本教程针对CH32H417/CH32H416系列微控制器的SAI（串行音频接口），从基础概念、核心功能、寄存器配置到实操流程，循序渐进讲解，帮助新手快速掌握SAI的使用方法，适配音频传输、协议通信等常见应用场景。教程结合芯片手册，聚焦实操重点，避开复杂理论冗余，兼顾入门易懂性和工程实用性。

# 第一章：SAI基础认知

## 1.1 什么是SAI

SAI（Serial Audio Interface，串行音频接口）是CH32H417系列内置的高速串行接口，专为音频传输设计，支持多种音频协议和传输模式，可实现高效、同步的音频数据收发。其核心优势在于灵活性高，可适配TDM、I2S、AC’97、SPDIF等多种音频协议，同时支持DMA传输和中断控制，能有效减轻CPU负担，广泛应用于音频播放器、语音采集、音频编解码等场景。

CH32H417的SAI包含两个独立的音频模块（A模块和B模块），每个模块均可独立配置为发送器或接收器，支持主模式、从模式，可满足单声道、立体声音频传输需求，还具备静音检测、数据压扩等实用功能。

## 1.2 核心特性总结（实操必记）

- 双独立模块：A、B两个音频模块，可独立工作，支持同步或异步模式；

- 多协议支持：自由协议（可自定义适配I2S、TDM等）、AC’97协议、SPDIF协议；

- 灵活配置：支持8/10/16/20/24/32位数据宽度，单声道/立体声切换；

- 高效传输：支持DMA传输和中断控制，减少CPU占用；

- 实用功能：静音检测、数据压扩（μ-Law/A-Law）、FIFO缓存、错误检测；

- 时钟灵活：支持主时钟（MCLK）配置，可通过分频器调整传输速率。

## 1.3 关键术语解释（避免混淆）

- SCK：串行时钟线，用于同步数据传输，由主模式设备生成；

- FS：帧同步信号，用于标识音频帧的起始和结束，决定帧长度和Slot分配；

- SD：串行数据线，用于传输音频数据（发送器输出SDO，接收器输入SDI）；

- Slot：时隙，一个音频帧由多个Slot组成，每个Slot对应一个声道或一段数据；

- FIFO：缓存区，用于暂存收发数据，避免数据丢失，支持阈值中断；

- 主模式/从模式：主模式生成SCK和FS信号，从模式跟随外部SCK和FS信号工作。

# 第二章：SAI核心功能与模式详解

SAI的核心功能围绕“协议配置”和“传输模式”展开，不同模式对应不同应用场景，实操中需根据需求选择合适的配置，以下是最常用的功能和模式详解。

## 2.1 核心工作模式（重点掌握）

### 2.1.1 自由协议模式（最常用）

自由协议是SAI最灵活的模式，可通过配置寄存器自定义帧长度、Slot数量、数据宽度等参数，适配I2S、TDM、PCM等常见音频协议，是大多数音频传输场景的首选。

核心配置要点：

- 协议配置：R32_SAI_xCFGR1寄存器PRTCFG[1:0] = 00b（自由协议）；

- 帧配置：通过R32_SAI_xFRCR寄存器设置帧长度（FRL）、帧同步极性（FSPOL）、帧同步偏移（FSOFF）；

- Slot配置：通过R32_SAI_xSLOTR寄存器设置Slot数量（NBSLOT）、Slot大小（SLOTSZ）、有效Slot（SLOTEN）；

- 数据配置：通过R32_SAI_xCFGR1寄存器设置数据宽度（DS[2:0]）、传输顺序（LSBFIRST）。

### 2.1.2 SPDIF模式（数字音频输出）

SPDIF（索尼/飞利浦数字接口）用于消费级音频设备的数字音频输出，仅支持发送模式，遵循IEC 60958标准，适用于音频信号的数字传输场景（如连接音响、功放）。

核心特性与配置：

- 协议配置：R32_SAI_xCFGR1寄存器PRTCFG[1:0] = 01b（SPDIF模式）；

- 强制配置：自动强制为主发送模式（MODE[1:0] = 00b），数据宽度强制为24位；

- 时钟要求：位时钟（SCK）应为符号率的两倍，符号率与音频采样频率对应（如48kHz采样率对应3.072MHz符号率）；

- 数据传输：数据通过R32_SAI_xDATAR寄存器写入，包含24位数据和4位状态位，采用曼彻斯特编码传输。

### 2.1.3 AC’97模式（音频链路控制）

AC’97模式用于AC’97协议的音频链路控制，适用于与AC’97编解码器通信，Slot数和Slot大小固定，帧同步信号波形固定，无需复杂配置。

核心特性：

- 协议配置：R32_SAI_xCFGR1寄存器PRTCFG[1:0] = 10b（AC’97模式）；

- 固定参数：Slot数固定为13，第一个Slot为16位，其余为20位；

- FS信号：无论主从模式，SAI自动生成FS信号（输出）；

- 特殊标志：支持CNRDY标志（编解码器未就绪），可触发中断。

## 2.2 关键辅助功能

### 2.2.1 单声道/立体声切换

仅当Slot数为2（R32_SAI_xSLOTR寄存器NBSLOT[3:0] = 0001b）时有效，通过R32_SAI_xCFGR1寄存器MONO位控制：

- MONO = 1：单声道模式，发送时Slot0数据复制到Slot1，接收时仅存储Slot0数据；

- MONO = 0：立体声模式，Slot0和Slot1分别对应左右声道。

### 2.2.2 静音模式

分为发送静音和接收静音检测，适用于音频启停控制和异常检测：

- 发送静音：通过R32_SAI_xCFGR2寄存器MUTE位置1使能，可选择发送0或上一个数据（MUTEVAL位控制）；

- 接收静音检测：通过R32_SAI_xCFGR2寄存器MUTECNT[5:0]设置连续静音帧数，检测到后MUTEDET标志置1，可触发中断。

### 2.2.3 数据压扩模式

适用于移动通信场景，支持μ-Law和A-Law两种压扩算法（CCITT G.711标准），仅在TDM协议下有效：

- 压扩配置：R32_SAI_xCFGR2寄存器COMP[1:0]选择算法（10b=μ-Law，11b=A-Law）；

- 补码设置：通过CPL位选择1的补码或2的补码计算；

- 数据宽度：使能压扩后，数据宽度强制为8位（DS[2:0] = 010b）。

### 2.2.4 错误检测与处理

SAI支持多种错误检测，便于排查传输异常，核心错误标志（R32_SAI_xSR寄存器）如下：

- OVRUDR：FIFO上溢/下溢（接收时FIFO满仍接收为上溢，发送时FIFO空仍发送为下溢）；

- AFSDET：帧同步提前检测（从模式下，提前检测到FS信号）；

- LFSDET：帧同步滞后检测（从模式下，未按时检测到FS信号）；

- CNRDY：编解码器未就绪（仅AC’97模式有效）；

- WCKCFG：主模式时钟配置错误（帧长度不符合规范）。

错误处理：所有错误标志均可通过“写1清零”，部分错误（如AFSDET、LFSDET）需重新使能SAI模块以恢复同步。

# 第三章：核心寄存器配置详解（实操必用）

CH32H417的SAI每个模块（A/B）包含7个核心寄存器，前缀分别为R32_SAI_A（A模块）和R32_SAI_B（B模块），以下重点讲解实操中最常用的寄存器，其余寄存器可参考官方手册补充。

## 3.1 配置寄存器1（R32_SAI_xCFGR1）

地址：A模块=0x40015804，B模块=0x40015824，复位值=0x00000040，核心配置位（仅列实操常用）：

|位字段|功能描述|实操配置要点|
|---|---|---|
|PRTCFG[1:0]（bit3~2）|协议配置：00=自由协议，01=SPDIF，10=AC’97，11=保留|根据应用场景选择，常规音频传输选00b|
|MODE[1:0]（bit1~0）|工作模式：00=主发送，01=主接收，10=从发送，11=从接收|SPDIF模式强制为00b；主模式需生成SCK/FS，从模式跟随外部信号|
|DS[2:0]（bit7~5）|数据宽度：010=8位，011=10位，100=16位，101=20位，110=24位，111=32位|根据音频数据规格选择，如常规音频选100b（16位）|
|SAIEN（bit16）|SAI模块使能：1=使能，0=禁止|所有配置完成后置1，禁用时置0；主模式需先有时钟再使能|
|DMAEN（bit17）|DMA使能：1=使能，0=禁止|需DMA传输时置1，配置前需确定工作模式（发送/接收）|
|MONO（bit12）|单声道模式：1=单声道，0=立体声|仅Slot数为2时有效，根据音频声道需求配置|
## 3.2 配置寄存器2（R32_SAI_xCFGR2）

地址：A模块=0x40015808，B模块=0x40015828，复位值=0x00000000，核心配置位：

|位字段|功能描述|实操配置要点|
|---|---|---|
|MUTE（bit5）|静音模式使能：1=使能，0=禁止（SPDIF模式无效）|音频启停时置1，需配合MUTEVAL位选择静音值|
|FFLUSH（bit3）|FIFO刷新：1=触发刷新，0=禁止|错误恢复或重新配置前，置1刷新FIFO，需先禁止DMA/中断|
|FTH[2:0]（bit2~0）|FIFO阈值：000=空，001=1/4，010=1/2，011=3/4，100=满|配合FREQ中断使用，如发送模式设001（1/4阈值）避免下溢|
|COMP[1:0]（bit15~14）|压扩模式：10=μ-Law，11=A-Law，其余无效|仅TDM协议使用，根据通信标准选择，无需压扩则置00b|
## 3.3 帧配置寄存器（R32_SAI_xFRCR）

地址：A模块=0x4001580C，B模块=0x4001582C，复位值=0x00000007，核心配置位（自由协议必配）：

- FRL[7:0]（bit7~0）：帧长度，帧位数=FRL+1，范围8~256，主模式下建议为2的幂次；

- FSPOL（bit17）：帧同步极性，1=高电平有效，0=低电平有效（0时需外接上拉）；

- FSOFF（bit18）：帧同步偏移，1=FS在Slot0第一位前触发，0=FS在Slot0第一位触发；

- FSDEF（bit16）：帧同步定义，1=FS为SOF+通道识别（Slot数需为偶数），0=FS为起始帧信号。

## 3.4 Slot寄存器（R32_SAI_xSLOTR）

地址：A模块=0x40015810，B模块=0x40015830，复位值=0x00000000，核心配置位（自由协议必配）：

- NBSLOT[3:0]（bit11~8）：Slot数，Slot数=NBSLOT+1，最大值16，FSDEF=1时需为偶数；

- SLOTSZ[1:0]（bit7~6）：Slot大小，00=与数据大小一致，01=16位，10=32位，需大于等于数据大小；

- SLOTEN[15:0]（bit31~16）：有效Slot，对应位=1表示该Slot有效，0表示无效；

- FBOFF[4:0]（bit4~0）：数据偏移，定义Slot中第一个数据位的位置，多余位发送时清零、接收时丢弃。

## 3.5 中断使能寄存器（R32_SAI_xINTENR）

地址：A模块=0x40015814，B模块=0x40015834，复位值=0x00000000，核心中断使能位：

- OVRUDRIE（bit0）：上溢/下溢中断使能，1=使能，0=禁止；

- FREQIE（bit3）：FIFO阈值中断使能，1=使能，0=禁止；

- AFSDETIE（bit5）：帧同步提前中断使能，1=使能，0=禁止；

- LFSDETIE（bit6）：帧同步滞后中断使能，1=使能，0=禁止。

## 3.6 状态寄存器（R32_SAI_xSR）

地址：A模块=0x40015818，B模块=0x40015838，复位值=0x00000008，核心状态位（只读，写1清零）：

- OVRUDR（bit0）：上溢/下溢标志，1=有错误，0=无错误；

- FREQ（bit3）：FIFO阈值标志，1=达到阈值，0=未达到；

- FLTH[2:0]（bit18~16）：FIFO当前级别，实时查看FIFO缓存状态；

- AFSDET（bit5）、LFSDET（bit6）：帧同步错误标志，1=有错误，0=无错误。

## 3.7 数据寄存器（R32_SAI_xDATAR）

地址：A模块=0x40015820，B模块=0x40015840，复位值=0x00000000，核心功能：

- 发送模式：FIFO未满时，写入该寄存器向FIFO加载数据；

- 接收模式：FIFO非空时，读取该寄存器获取接收数据；

- SPDIF模式：写入24位数据+4位状态位，数据存储在DATA[23:0]，状态位在DATA[26:24]。

# 第四章：SAI实操流程（以自由协议+主发送为例）

以最常用的“自由协议+主发送+16位立体声”为例，讲解SAI的完整实操流程，适配常规音频传输场景，其他模式可参考此流程调整配置。

## 4.1 实操准备

- 硬件连接：将SAI的SCK、FS、SDO引脚与外部设备（如编解码器、音响）对应连接；

- 参数确定：音频采样率48kHz，数据宽度16位，立体声（2个Slot），帧长度32位（FRL=31）；

- 时钟配置：HCLK=144MHz，主时钟（MCLK）=12.288MHz，SCK=3.072MHz（48kHz×64）。

## 4.2 初始化步骤（核心代码逻辑，以A模块为例）

1. 禁止SAI模块，等待空闲：

    - R32_SAI_ACFGR1 &= ~(1 << 16); // 清SAIEN位，禁止A模块

    - while((R32_SAI_ASR & (1 << 3)) == 0); // 等待FREQ标志为1（FIFO空闲）

2. 配置帧参数（R32_SAI_AFRCR）：
        

    - R32_SAI_AFRCR = 0x0000001F; // FRL=31（帧长度32位），FSPOL=0（低电平有效），FSDEF=1（通道识别）

3. 配置Slot参数（R32_SAI_ASLOTR）：
        

    - R32_SAI_ASLOTR = 0x00000001; // NBSLOT=1（2个Slot），SLOTSZ=00（与数据大小一致）

    - R32_SAI_ASLOTR |= 0x00000003; // SLOTEN[1:0]=11（两个Slot均有效）

4. 配置控制参数（R32_SAI_ACFGR1）：
        

    - R32_SAI_ACFGR1 = 0x00000000; // PRTCFG=00（自由协议），MODE=00（主发送）

    - R32_SAI_ACFGR1 |= (0x04 << 5); // DS=100（16位数据）

    - R32_SAI_ACFGR1 |= (1 << 17); // DMAEN=1（使能DMA）

    - R32_SAI_ACFGR1 |= (0x01 << 12); // MONO=0（立体声）

5. 配置FIFO阈值（R32_SAI_ACFGR2）：


    - R32_SAI_ACFGR2 = 0x00000001; // FTH=001（1/4阈值，避免下溢）

6. 使能中断（可选）：
       

    - R32_SAI_AINTENR |= (1 << 0) | (1 << 3); // 使能上溢/下溢、FIFO阈值中断

7. 使能SAI模块：
        

    - R32_SAI_ACFGR1 |= (1 << 16); // 置SAIEN位，使能A模块

## 4.3 数据发送操作（DMA模式）

1. 配置DMA通道：选择SAI_A发送DMA通道，设置数据长度、源地址（音频数据缓冲区）、目标地址（R32_SAI_ADATAR）；

2. 启动DMA传输：使能DMA通道，开始将音频数据写入SAI FIFO；

3. 状态监测：通过R32_SAI_ASR寄存器查看FREQ、OVRUDR标志，判断传输状态；

4. 传输完成：DMA传输完成后，禁止DMA通道，可触发中断处理后续逻辑（如切换音频数据）。

## 4.4 错误处理流程

若检测到OVRUDR（上溢/下溢）、AFSDET（帧同步提前）等错误，处理步骤如下：

1. 禁止SAI模块和DMA通道；

2. 清零错误标志：R32_SAI_ASR |= (1 << 0) | (1 << 5) | (1 << 6); // 清零OVRUDR、AFSDET、LFSDET

3. 刷新FIFO：R32_SAI_ACFGR2 |= (1 << 3); // 置FFLUSH位，刷新FIFO

4. 重新配置SAI寄存器（可复用初始化参数）；

5. 重新使能SAI和DMA，恢复传输。

# 第五章：常见问题与排查技巧

## 5.1 常见问题及解决方案

|问题现象|可能原因|解决方案|
|---|---|---|
|无数据传输|1. SAI模块未使能；2. 时钟配置错误；3. Slot未使能|1. 检查SAIEN位；2. 重新配置MCLK/分频器；3. 确认SLOTEN位|
|OVRUDR标志置1|1. FIFO阈值设置不合理；2. DMA传输速率不匹配；3. 数据长度错误|1. 调整FTH阈值；2. 优化DMA配置；3. 核对DLR寄存器|
|帧同步错误（AFSDET/LFSDET）|1. 从模式下FS信号异常；2. 帧长度配置错误；3. 时钟同步异常|1. 检查FS引脚连接；2. 核对FRL参数；3. 确保主从时钟同步|
|SPDIF无输出|1. 协议配置错误；2. 时钟速率不匹配；3. 数据格式错误|1. 确认PRTCFG=01b；2. 调整SCK速率；3. 按24位数据+4位状态位写入|
## 5.2 实操排查技巧

- 寄存器排查：先检查SAIEN位是否使能，再核对PRTCFG、MODE、DS等核心配置位；

- 状态位排查：通过R32_SAI_xSR寄存器的FLTH位查看FIFO状态，通过错误标志定位问题；

- 硬件排查：检查SCK、FS、SD引脚连接是否正确，电源是否稳定，外部设备是否正常；

- 简化测试：先使用查询模式（不启用DMA），确保数据能正常收发后，再启用DMA和中断。
