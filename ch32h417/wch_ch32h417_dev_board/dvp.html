CH32H417系列开发板使用手册

CH32H417 数字图像接口（DVP）实操教程

本教程基于CH32H417系列MCU，详细讲解数字图像接口（DVP）的核心原理、主要特性、功能描述、实操配置及寄存器使用方法，结合传感器连接、工作时序、捕获模式等关键知识点，帮助开发者快速掌握DVP接口的使用，实现与摄像头模块的稳定连接及图像数据的高效采集。

第一章：DVP接口基础认知

1.1 什么是DVP接口

数字图像接口DVP（Digital Video Port）是CH32H417 MCU用于连接摄像头模块、获取图像数据流的核心接口，采用8/10/12位并行接口方式通信，支持最高150MHz像素时钟输入频率，可接收原始格式（YUV、RGB等）和压缩格式（JPEG）的图像数据，依靠VSYNC（帧同步）和HSYNC（行同步）信号实现数据同步，同时支持图像裁剪功能，适配各类高速摄像头模块的图像采集需求。

1.2 核心特性（必记）

  • 数据宽度可配置：支持8位、10位、12位三种并行接口模式，适配不同规格的摄像头模块。

  • 数据格式支持：兼容YUV、RGB等原始图像格式，同时支持JPEG压缩格式数据，灵活适配不同采集需求。

  • 高速传输能力：内置FIFO缓冲区，支持DMA传输，可接收最高150MHz像素时钟的高速并行数据流。

  • 双缓冲接收：采用BUF0和BUF1双缓冲交替存储数据，提升数据接收的稳定性和效率。

  • 灵活捕获模式：支持连续模式（持续采集帧数据）和快照模式（仅采集单帧数据）。

  • 图像裁剪功能：可通过配置寄存器，截取图像中的指定矩形窗口，满足个性化采集需求。

第二章：DVP接口功能详解

2.1 与摄像头传感器的连接

DVP接口通过多组信号与摄像头传感器（Sensor）建立连接，实现时钟、同步信号和数据的传输，同时通过I2C接口配置传感器参数，连接示意及各信号说明如下：

2.1.1 连接信号说明

  • PCLK（Pixel clk）：像素时钟，每个时钟周期对应一个非压缩像素数据，外部传感器输出的PCLK时钟最大支持150MHz。

  • HSYNC（Horizontal Synchronization）：行同步信号，用于同步图像的每一行数据传输，标识一行数据的起始和结束。

  • VSYNC（Vertical Synchronization）：帧同步信号，用于同步一帧完整图像的数据传输，标识一帧图像的起始和结束。

  • DATA[11:0]：像素数据总线，用于传输像素数据（原始格式或压缩格式），位宽可配置为8/10/12位，对应D0~D11引脚。

  • XCLK：传感器参考时钟，可由MCU提供或外部晶振提供，为摄像头传感器的正常工作提供时钟基准。

  • I2C接口（SCL、SDA）：用于配置传感器的寄存器，可通过MCU的普通GPIO口模拟I2C时序，也可使用硬件I2C接口操作。

2.1.2 DVP引脚汇总

引脚名称信号类型及描述
PCLK像素时钟输入，最大支持150MHz
DATA[11:0]像素数据输入，位宽可配置为8/10/12位
VSYNC帧同步信号输入，标识一帧图像的起始/结束
HSYNC行同步信号输入，标识一行数据的起始/结束

2.2 工作时序解析

DVP接口的工作时序与摄像头传感器输出的图像大小、刷新率密切相关，核心是通过PCLK、HSYNC、VSYNC三个信号的配合，实现有效图像数据的采集，以下结合实际示例详细说明：

2.2.1 时序核心逻辑

以某传感器为例，传感器内部完整图像尺寸为3576×1145，经内部缩放后，输出图像尺寸为1920×1080，刷新率为25帧/秒（即每秒采集25帧图像），时序关系如下：

  • 帧时序：1帧（1 frame cycle）包含1145行（1145 row cycles），25帧对应1秒（25 frame cycles = 1 second）。

  • 行时序：1行（1 row cycle）包含3576个PCLK时钟周期（1 row cycle = 3576 PCLKs），其中仅1920个PCLK周期传输有效像素数据，剩余1656个PCLK周期传感器不传输数据。

  • 同步信号关系：HSYNC信号周期是PCLK的3576倍（对应1行数据），VSYNC信号周期是PCLK的3576×1145倍（对应1帧数据）。

2.2.2 特殊说明

若传感器传输的是JPEG压缩格式数据，由于数据为压缩后的连续数据流，无需依赖HSYNC行同步信号，仅需通过VSYNC帧同步信号和PCLK时钟即可完成数据接收，具体时序需以所选用传感器的数据手册为准。

2.3 图像数据格式说明

DVP接口支持原始图像格式和压缩图像格式，三种常用格式的核心特点如下：

  • RGB格式：基于三原色（红色、绿色、蓝色）组成，每个像素由RGB三个分量构成，属于原始未压缩格式，色彩还原度高，适用于对图像质量要求较高的场景。

  • YUV格式：包含亮度信号（Y）和色度信号（U、V，或Pb、Pr，或Cb、Cr），亮度与色度分离，兼顾图像质量和传输效率，是常用的原始图像格式之一。

  • JPEG格式：一种有损失压缩格式，利用人眼对图像高频信息不敏感的特性，去除视觉冗余和结构冗余，压缩后数据量小，适用于对数据传输速率要求较高、对图像细节要求不极致的场景。

第三章：DVP接口实操配置（重点）

DVP接口的核心应用是接收摄像头传感器输出的图像数据，实操配置需严格遵循“寄存器配置→模式选择→数据接收”的流程，确保配置参数与传感器输出匹配，以下是详细步骤及说明（以CH32H417为例）。

3.1 配置核心说明

  • 双缓冲存储：DVP接收的每帧数据由BUF0和BUF1交替存储，默认从BUF0开始；RGB/YUV格式下，帧同步信号（VSYNC）有效时复位选择BUF0，存满一行切换至BUF1；JPEG格式下，按DMA接收长度设置切换阈值。

  • 数据扩展：当数据总线宽度为10位或12位时，系统会自动将接收的数据无符号扩展至16位后存储，无需手动配置。

  • 寄存器匹配：R16_DVP_ROW_NUM（行数配置）和R16_DVP_COL_NUM（列数配置）必须与传感器实际输出的图像大小匹配，否则会导致数据接收异常。

  • 模式差异：RGB模式下，R16_DVP_COL_NUM表示一行有效PCLK周期数，R16_DVP_ROW_NUM表示一帧有效行数；JPEG模式下，仅R16_DVP_COL_NUM用于配置DMA接收长度，R16_DVP_ROW_NUM无效。

3.2 完整配置步骤（实操必做）

使用DVP接口接收图像数据时，需按以下步骤配置相关寄存器，确保与摄像头传感器模式匹配：

  1. 复位接收逻辑：通过R8_DVP_CR1寄存器，清除RB_DVP_ALL_CLR（标志与FIFO清除）和RB_DVP_RCV_CLR（接收逻辑复位）字段，复位接收相关逻辑和FIFO缓冲区，避免残留数据影响采集。

  2. 配置基础参数：通过R8_DVP_CR0寄存器，配置图像模式（原始/RPEG）、数据位宽（8/10/12位）、PCLK极性、HSYNC极性和VSYNC极性，确保与传感器输出的信号极性、数据格式一致。

  3. 配置图像尺寸：根据传感器实际输出的有效图像像素，配置R16_DVP_ROW_NUM（行数）和R16_DVP_COL_NUM（列数）；若为JPEG模式，仅配置R16_DVP_COL_NUM即可。

  4. 配置DMA接收地址：通过R32_DVP_DMA_BUF0和R32_DVP_DMA_BUF1寄存器，分别配置双缓冲的DMA接收地址，要求地址为32字节对齐，确保DMA传输正常。

  5. 选择捕获模式（可选）：

    • 快照模式：通过R8_DVP_CR1寄存器的RB_DVP_CM位置1，开启单帧捕获模式，捕获一帧数据后自动关闭DVP接口。

    • 连续模式：默认模式（RB_DVP_CM位清0），开启DVP接口后持续采集每帧数据，直至手动关闭DVP接口。

  6. 开启裁剪功能（可选）：通过R8_DVP_CR1寄存器的RB_DVP_CROP（裁剪使能）和RB_DVP_FCRC（帧捕获率控制）字段，开启裁剪功能并设置帧捕获率；同时配置R16_DVP_HOFFCNT（水平偏移）、R16_DVP_VST（起始行数）、R16_DVP_CAPCNT（水平捕获数）、R16_DVP_VLINE（垂直捕获数），设置裁剪窗口的坐标和大小。

  7. 配置中断（可选）：根据需求，通过R8_DVP_IER寄存器使能对应中断（帧开始、行结束、帧完成、FIFO溢出、帧结束），并通过中断控制器PFIC配置中断优先级，开启DVP中断，实现数据接收的中断触发处理。

  8. 使能DVP和DMA：通过R8_DVP_CR1寄存器的RB_DVP_DMA_ENABLE位置1，使能DMA传输；通过R8_DVP_CR0寄存器的RB_DVP_ENABLE位置1，使能DVP接口，开始接收图像数据。

  9. 数据处理：等待相关接收中断产生，在中断服务函数中处理接收的数据（如存储、解析、显示等），确保数据及时读取，避免FIFO溢出。

3.3 捕获模式详解

3.3.1 快照模式（单帧捕获）

  • 配置方式：R8_DVP_CR1寄存器中RB_DVP_CM位置1，开启快照模式。

  • 工作逻辑：使能DVP接口后，系统等待VSYNC帧同步信号（帧起始），开始采集单帧图像数据；接收完一帧完整数据后，系统自动将R8_DVP_CR0寄存器的RB_DVP_ENABLE位清0，关闭DVP接口，完成单帧捕获。

3.3.2 连续模式（持续捕获）

  • 配置方式：R8_DVP_CR1寄存器中RB_DVP_CM位清0，默认开启连续模式。

  • 工作逻辑：使能DVP接口后，系统持续采集每帧图像数据，直至手动将R8_DVP_CR0寄存器的RB_DVP_ENABLE位清0，关闭DVP接口；期间每帧数据通过双缓冲交替存储，配合DMA传输，确保数据不丢失。

3.4 裁剪功能详解

DVP接口的裁剪功能可从接收的完整图像中，截取指定大小的矩形窗口数据，适配不同的图像显示或处理需求，核心配置如下：

  • 裁剪窗口参数：

    • 起始坐标：由R16_DVP_HOFFCNT（水平偏移，矩形左上角X坐标）和R16_DVP_VST（起始行数，矩形左上角Y坐标）配置。

    • 窗口大小：由R16_DVP_CAPCNT（水平捕获数，矩形宽度）和R16_DVP_VLINE（垂直捕获数，矩形高度）配置。

  • 开启方法：通过R8_DVP_CR1寄存器的RB_DVP_CROP位置1，开启裁剪功能；同时配置RB_DVP_FCRC字段，控制帧捕获率（00=捕获所有帧，01=每隔一帧捕获，10=每隔三帧捕获，11=保留）。

  • 时序特点：裁剪时，HSYNC行同步信号周期不变，仅在每行的有效PCLK周期内，截取指定长度的像素数据；VSYNC帧同步信号周期不变，仅截取指定行数的有效数据。

第五章：实操注意事项

  • 参数匹配：DVP接口的位宽、信号极性、图像尺寸等配置，必须与摄像头传感器的输出参数完全一致，否则会导致数据接收异常、图像失真或无法接收数据。

  • 时钟控制：PCLK时钟最大支持150MHz，配置时需结合传感器的时钟要求，避免时钟频率过高导致数据传输错误；XCLK参考时钟需与传感器的工作时钟匹配，优先使用晶振提供。

  • DMA配置：DMA接收地址需满足32字节对齐要求，否则会导致DMA传输失败；双缓冲BUF0和BUF1的地址需独立配置，避免地址重叠。

  • 中断处理：开启中断后，需及时处理中断标志（通过R8_DVP_IFR寄存器写1清除），避免中断反复触发；FIFO溢出中断需重点关注，及时读取数据防止溢出。

  • 裁剪配置：裁剪窗口的坐标和大小需在传感器输出图像尺寸范围内，否则会截取到无效数据；低14位/13位有效位需注意，避免配置超出有效范围。

  • 传感器配置：通过I2C接口配置传感器寄存器时，需确保I2C时序正确，可优先使用硬件I2C接口，提升配置稳定性。
