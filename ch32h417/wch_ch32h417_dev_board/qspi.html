CH32H417/CH32H416 QSPI（QuadSPI）接口实操教程

本教程专门针对CH32H417、CH32H416微控制器的QSPI（QuadSPI）接口编写，基于官方应用手册，省略冗余理论，聚焦实操核心，涵盖QSPI接口的功能特性、工作模式、命令序列、寄存器配置及实操流程，助力开发者快速掌握QSPI接口的使用方法，适配单/双/四线SPI FLASH的连接与控制。

第一章：QSPI接口基础认知

1.1 接口概述

CH32H417/CH32H416芯片内置两组独立的QSPI通信接口，核心功能是连接单、双或四线SPI FLASH存储介质，支持多种工作模式，可满足不同场景下的FLASH读写、状态监控及内存映射需求。其核心优势在于高速传输、灵活的模式配置，以及双闪存模式下的吞吐量和容量翻倍能力，广泛应用于需要大量数据存储与快速访问的场景。

1.2 核心功能模式

QSPI接口支持4种核心工作模式，适配不同应用场景，具体如下：

  • 间接模式：通过配置QSPI相关寄存器执行所有操作（读写、指令发送等），适用于需要频繁进行FLASH读写的应用场景，是最常用的工作模式。

  • 状态轮询模式：系统周期性读取外部FLASH的状态寄存器，当目标标志位置1（如擦除、烧写完成）时，触发中断通知系统，适用于需要实时监控FLASH工作状态的场景。

  • 内存映射模式：将外部FLASH直接映射到微控制器的地址空间，系统可将其视为内部存储器进行访问（仅支持数据访问，不支持指令访问），适用于需要快速读取FLASH数据的场景。

  • 双闪存模式：可同时访问两个Quad-SPI FLASH，吞吐量和存储容量均翻倍，适用于需要大量数据处理、高吞吐量的应用场景。

1.3 主要特征

  • 支持间接模式、状态轮询模式、内存映射模式三种核心功能模式，可灵活切换；

  • 支持双闪存模式，并行访问两个FLASH，提升吞吐量和存储容量；

  • 集成32字节FIFO，用于数据发送和接收，提升传输效率，支持FIFO阈值中断；

  • 仅支持SDR（单倍数据速率）模式，时序稳定，适配多数SPI FLASH；

  • 间接模式和内存映射模式下，支持完全可编程的帧格式和操作码；

  • 支持DMA触发（FIFO阈值、传输完成时）和多类型中断（超时、状态匹配、传输错误等）；

  • 支持单线、双线、四线SPI通信模式，可根据FLASH规格灵活配置。

1.4 功能框图解析

QSPI接口的核心结构围绕“总线连接-控制模块-FIFO-移位寄存器-外部接口”展开，分为单闪存模式和双闪存模式：

  • 单闪存模式（双闪存模式禁止）：通过32位HB总线与内核通信，核心模块包括控制寄存器、时钟管理、FIFO、移位寄存器，外部接口为SCK（时钟）、SIO0~SIO3（数据线）、SCSN（片选），连接单个SPI FLASH。

  • 双闪存模式（双闪存模式使能）：在单闪存模式基础上，扩展一组SIOX0~SIOX3数据线和SCSXN片选信号，可同时连接两个SPI FLASH，共享SCK时钟，实现并行数据传输。

核心信号说明：SCK为时钟信号，SIO0~SIO3（双闪存模式下增加SIOX0~SIOX3）为数据I/O线，SCSN（双闪存模式下增加SCSXN）为片选信号（低电平有效）。

第二章：QSPI核心功能详解

2.1 QSPI命令序列（核心实操基础）

QSPI与外部FLASH通信的核心是命令序列，每条命令包含5个可配置阶段（任一阶段可跳过，至少保留一个阶段），且nCS（片选）在命令开始前拉低，命令完成后拉高。5个阶段依次为：

2.1.1 指令阶段

用于向FLASH发送8位操作指令（如读、写、擦除），指令配置在R32_QSPIx_CCR寄存器的INSTRUCTION[7:0]字段。

  • 传输模式：通过R32_QSPIx_CCR寄存器的IMODE[1:0]字段配置，可选“无指令（00b）、单线传输（01b）、双线传输（10b）、四线传输（11b）”；

  • 注意：多数FLASH仅支持单线模式接收指令，需根据FLASH规格配置IMODE。

2.1.2 地址阶段

用于指定FLASH的操作地址（如读/写地址），可配置1~4字节地址，核心配置如下：

  • 地址长度：通过R32_QSPIx_CCR寄存器的ADSIZE[1:0]字段配置（00b=8位，01b=16位，10b=24位，11b=32位）；

  • 传输模式：通过R32_QSPIx_CCR寄存器的ADMODE[1:0]字段配置（同指令阶段，00b=无地址）；

  • 地址指定：间接模式/状态轮询模式下，地址存于R32_QSPIx_AR寄存器的ADDRESS[31:0]；内存映射模式下，地址由系统直接提供。

2.1.3 交替字节阶段

用于向FLASH发送控制配置数据（通常用于控制FLASH工作模式），可配置1~4字节，核心配置如下：

  • 交替字节长度：通过R32_QSPIx_CCR寄存器的ABSIZE[1:0]字段配置（同地址长度）；

  • 传输模式：通过R32_QSPIx_CCR寄存器的ABMODE[1:0]字段配置（同指令阶段，00b=无交替字节）；

  • 数据指定：交替字节存于R32_QSPIx_ABR寄存器的ALTERNATE[31:0]；

  • 特殊场景：若只需发送半字节，可配置ABMODE=11b（四线模式），通过ALTERNATE寄存器的特定位传输。

2.1.4 空指令周期阶段

用于在高频时钟下，给FLASH足够的时间准备数据，核心配置如下：

  • 周期数：通过R32_QSPIx_CCR寄存器的DCYC[4:0]字段配置（0~31个SCK周期），DCYC=0时跳过该阶段；

  • 传输模式：固定为单线模式，与其他阶段模式无关。

2.1.5 数据阶段

用于与FLASH进行数据交换（读/写），可传输任意数量字节，核心配置如下：

  • 传输模式：通过R32_QSPIx_CCR寄存器的DMODE[1:0]字段配置（00b=无数据，仅适用于间接写入模式）；

  • 数据长度：间接模式/状态轮询模式下，由R32_QSPIx_DLR寄存器的DL[31:0]字段指定（值+1为实际字节数），全1表示未定义长度（传输至FLASH末尾）；

  • 数据传输：间接写入时，数据写入R32_QSPIx_DR寄存器，存入FIFO后发送；间接读取时，数据从FLASH接收后存入FIFO，通过读取R32_QSPIx_DR获取；内存映射模式下，数据直接返回系统。

2.2 QSPI信号接口协议模式

QSPI支持单线、双线、四线三种SPI协议模式，可根据FLASH的通信规格灵活配置，三种模式的核心区别的在于数据传输的I/O线数量和信号行为：

2.2.1 单线SPI模式

  • 配置：将各阶段（指令/地址/交替字节/数据）的模式字段（IMODE/ADMODE/ABMODE/DMODE）设为01b；

  • 信号行为：SIO0为输出（发送数据），SIO1为输入（接收数据），SIO2强制置0（禁用写保护），SIO3强制置1（禁用保持）；

  • 适用场景：传统SPI FLASH，传输速率较低，兼容性强。

2.2.2 双线SPI模式

  • 配置：将各阶段模式字段设为10b；

  • 信号行为：SIO0/SIO1在数据读取时为输入（高阻态），其他阶段为输出；SIO2强制置0，SIO3强制置1；

  • 适用场景：需要提升传输速率，且FLASH支持双线通信的场景。

2.2.3 四线SPI模式

  • 配置：将各阶段模式字段设为11b；

  • 信号行为：SIO0~SIO3在数据读取时为输入（高阻态），其他阶段为输出；若未配置四线模式，SIO2/SIO3可用于其他功能；

  • 适用场景：高速传输场景，是QSPI的核心通信模式。

2.2.4 SDR模式（单倍数据速率）

QSPI仅支持SDR模式，核心时序特性：

  • 发送数据：QSPI驱动SIO0~SIO3信号时，仅在SCK的下降沿发生电平变化；

  • 接收数据：QSPI在SCK的上升沿采样FLASH发送的数据。

2.2.5 双闪存模式

通过配置R32_QSPIx_CR寄存器的DFM位（置1）开启，核心特性如下：

  • 连接两个SPI FLASH，共享SCK时钟，可共用片选信号（SCSN），数据线独立（SIO0~SIO3对应FLASH1，SIOX0~SIOX3对应FLASH2）；

  • 吞吐量翻倍：每个周期传输8位数据（两个FLASH各传输4位）；

  • 地址分配：偶地址对应FLASH1，奇地址对应FLASH2，需确保访问字节数为偶数；

  • 状态读取：读取状态寄存器时，需读取2倍于单闪存模式的字节数（两个FLASH各返回1份状态数据）。

2.3 三种核心工作模式详解（实操重点）

2.3.1 间接模式（最常用）

核心：通过配置QSPI寄存器触发命令，数据通过FIFO和数据寄存器（R32_QSPIx_DR）传输，类似普通SPI外设的工作方式，分为间接写入和间接读取两种子模式。

（1）模式配置

  • 间接写入模式：R32_QSPIx_CCR寄存器FMODE[1:0] = 00b，用于向FLASH写入数据；

  • 间接读取模式：R32_QSPIx_CCR寄存器FMODE[1:0] = 01b，用于从FLASH读取数据。

（2）命令启动触发方式

配置完相关寄存器后，通过以下三种方式之一触发命令启动（根据是否需要地址、数据判断）：

  1. 无需地址（ADMODE=00b）且无需数据（FMODE=01b或DMODE=00b）：写入R32_QSPIx_CCR寄存器的INSTRUCTION字段；

  2. 需要地址（ADMODE≠00b）且无需数据（FMODE=01b或DMODE=00b）：写入R32_QSPIx_AR寄存器的ADDRESS字段；

  3. 需要地址（ADMODE≠00b）且需要数据（FMODE=00b且DMODE≠00b）：写入R32_QSPIx_DR寄存器的DATA字段。

（3）FIFO与数据管理

  • FIFO容量：32字节，通过R32_QSPIx_SR寄存器的FLEVEL[5:0]字段查看当前FIFO中的字节数；

  • FIFO阈值：通过R32_QSPIx_CR寄存器的FTHRES[4:0]字段配置，达到阈值时FTF标志置1，可触发中断；

  • 写入逻辑：间接写入时，向R32_QSPIx_DR写入数据（字/半字/字节），自动存入FIFO，FIFO满时暂停写入；

  • 读取逻辑：间接读取时，从R32_QSPIx_DR读取数据，FIFO空时暂停读取，直到有新数据或传输完成。

2.3.2 状态轮询模式

核心：周期性发送命令读取FLASH状态寄存器，通过屏蔽和匹配逻辑检测目标状态，触发中断或停止轮询，适用于监控FLASH擦除、烧写完成状态。

（1）模式配置

R32_QSPIx_CCR寄存器FMODE[1:0] = 10b，同时配置命令序列（指令、地址等），数据长度（R32_QSPIx_DLR）最大为4（读取4字节状态数据）。

（2）核心配置步骤

  1. 配置R32_QSPIx_PSMKR寄存器（屏蔽寄存器）：MASK[n] = 1时，不屏蔽该位，参与匹配；MASK[n] = 0时，屏蔽该位；

  2. 配置R32_QSPIx_PSMAR寄存器（匹配寄存器）：设置需要匹配的状态位值；

  3. 配置匹配模式：R32_QSPIx_CR寄存器PMM位（0=AND模式，所有未屏蔽位匹配才置位；1=OR模式，任一未屏蔽位匹配即置位）；

  4. 配置轮询间隔：R32_QSPIx_PIR寄存器的INTERVAL[15:0]字段，指定两次状态读取之间的SCK周期数；

  5. 启动轮询：配置完地址（若需要）后，自动启动周期性读取，BUSY位置1。

（3）中断与停止

  • 状态匹配时，SMF标志置1，若SMIE=1（R32_QSPIx_CR），触发中断；

  • 若APMS=1（R32_QSPIx_CR），状态匹配后自动停止轮询，BUSY位清零；否则持续轮询，直到中止或禁用QSPI。

2.3.3 内存映射模式

核心：将外部FLASH映射到微控制器地址空间，视为内部存储器访问（仅支持读操作），无需通过寄存器读写，直接通过地址访问，适用于高速读取场景。

（1）模式配置

R32_QSPIx_CCR寄存器FMODE[1:0] = 11b，同时配置命令序列（指令、地址等），数据长度寄存器（R32_QSPIx_DLR）无效。

（2）核心特性

  • 寻址限制：可寻址空间最大256MB，仅支持数据访问，不支持指令访问；

  • 预取功能：QSPI会提前加载后续地址的数据到FIFO，加速连续地址访问；

  • 低功耗优化：通过R32_QSPIx_CR寄存器的TCEN位（置1）开启超时计数器，FIFO写满后，若超时（TIMEOUT[15:0]配置）无访问，释放nCS，降低FLASH功耗；

  • BUSY位：首次访问后，BUSY位置1，仅在超时、中止或禁用QSPI后清零。

2.4 其他关键功能

2.4.1 指令仅发送一次（SIOO位）

部分FLASH支持“首次命令发送指令，后续命令无需发送”的模式，通过R32_QSPIx_CCR寄存器的SIOO位（置1）实现：

  • SIOO=1时，仅第一条命令发送指令（写入R32_QSPIx_CCR时），后续命令跳过指令阶段，直到再次写入R32_QSPIx_CCR；

  • IMODE=00b（无指令）时，SIOO位无效。

2.4.2 差错管理

QSPI支持传输错误检测，当出现以下情况时，TEF标志（R32_QSPIx_SR）置1，可触发中断（TEIE=1）：

  • 间接模式/状态轮询模式下，R32_QSPIx_AR寄存器配置的地址超出R32_QSPIx_DCR寄存器FSIZE定义的FLASH大小；

  • 间接模式下，地址+数据长度超出FLASH大小。

2.4.3 繁忙位与中止功能

  • BUSY位（R32_QSPIx_SR）：QSPI启动操作后置1，操作完成且FIFO为空后清零（间接模式）；状态轮询模式下，仅在匹配停止、中止或禁用QSPI后清零；内存映射模式下，仅在超时、中止或禁用后清零；

  • 中止功能：通过R32_QSPIx_CR寄存器的ABORT位（置1）中止当前操作，中止完成后，ABORT位、BUSY位自动清零，FIFO清空。

2.4.4 nCS行为

nCS（片选）默认高电平（未选中FLASH），命令开始前拉低，命令完成后拉高，具体行为由CKMODE位（R32_QSPIx_DCR）配置：

  • CKMODE=0（模式0）：无操作时SCK保持低电平，nCS在首次SCK上升沿前1个周期拉低，最后一次SCK上升沿后1个周期拉高；

  • CKMODE=1（模式3）：无操作时SCK保持高电平，nCS行为与模式0一致；

  • 中止时：nCS在中止请求后立即拉高，SCK在半个周期后拉高。

第三章：核心寄存器配置详解（实操必用）

CH32H417/CH32H416的QSPI接口包含13个核心寄存器（QSPI1和QSPI2寄存器地址不同，前缀分别为R32_QSPI1_和R32_QSPI2_），以下重点讲解实操中最常用的寄存器，其余寄存器可参考官方手册补充。

3.1 控制寄存器（R32_QSPIx_CR）

地址：QSPI1=0x40024C00，QSPI2=0x40025000，复位值=0x00000000，核心配置位如下（仅列出实操常用位）：

位字段功能描述实操配置要点
PRESCALER[7:0]时钟预分频器，分频系数=值+1，用于生成SCK时钟根据HCLK频率和FLASH规格配置，如0x0=SCK=HCLK，0x1=SCK=HCLK/2
PMM（bit23）轮询匹配模式，0=AND模式，1=OR模式状态轮询模式下配置，根据需求选择匹配逻辑
APMS（bit22）自动轮询模式停止，1=匹配时停止轮询状态轮询模式下，需自动停止时置1
TOIE/SMIE/FTIE/TCIE/TEIE（bit20~16）分别为超时、状态匹配、FIFO阈值、传输完成、传输错误中断使能需要对应中断时置1，否则置0
FTHRES[4:0]（bit12~8）FIFO阈值，定义FTF标志置1的字节数间接模式下配置，根据传输效率需求设置（如00001=2字节阈值）
DFM（bit6）双闪存模式使能，1=使能，0=禁止需要同时访问两个FLASH时置1，否则置0
START（bit5）启动FLASH命令序列，写1触发，硬件自动清零间接模式/状态轮询模式下，配置完寄存器后写1启动
TCEN（bit3）超时计数器使能，仅内存映射模式有效内存映射模式下，需低功耗优化时置1
DMAEN（bit2）DMA模式使能，1=使能，0=禁止需要DMA传输数据时置1
ABORT（bit1）中止请求/清空FIFO，写1触发需要中止当前操作或清空FIFO时置1
EN（bit0）QSPI使能，1=使能，0=禁止所有配置完成后置1，禁用QSPI时置0

3.2 器件配置寄存器（R32_QSPIx_DCR）

地址：QSPI1=0x40024C04，QSPI2=0x40025004，复位值=0x00000000，核心配置位：

位字段功能描述实操配置要点
FSIZE[4:0]（bit20~16）FLASH大小，FLASH字节数=2^(FSIZE+1)，双闪存模式下为两个FLASH总容量根据所接FLASH容量配置，如16MB FLASH（2^24），FSIZE=23
CSHT[2:0]（bit10~8）片选高电平时间，CSHT+1为命令之间nCS保持高电平的SCK周期数根据FLASH规格配置，默认000（1个周期）
CKMODE（bit0）时钟模式，0=模式0，1=模式3与外部FLASH的时钟模式匹配，多数FLASH支持模式0或模式3

3.3 状态寄存器（R32_QSPIx_SR）

地址：QSPI1=0x40024C08，QSPI2=0x40025008，复位值=0x00000000，核心状态位（只读）：

位字段功能描述实操判断要点
IDLEF（bit16）空闲标志，1=QSPI总线空闲配置寄存器前，需确保IDLEF=1（BUSY=0）
FLEVEL[5:0]（bit13~8）FIFO级别，指示当前FIFO中的有效字节数间接模式下，判断FIFO是否有数据/空闲空间
BUSY（bit5）忙标志，1=QSPI正在执行操作配置寄存器、启动命令前，需确保BUSY=0
TOF（bit4）超时标志，1=发生超时内存映射模式下，超时后需通过FCR寄存器清零
SMF（bit3）状态匹配标志，1=状态匹配成功状态轮询模式下，判断是否达到目标状态，需通过FCR寄存器清零
FTF（bit2）FIFO阈值标志，1=达到FIFO阈值间接模式下，触发FIFO阈值中断，无需手动清零（阈值条件消失后自动清零）
TCF（bit1）传输完成标志，1=传输完成或中止间接模式下，判断传输是否完成，需通过FCR寄存器清零
TEF（bit0）传输错误标志，1=发生传输错误出现错误时查看，需通过FCR寄存器清零

3.4 标志清零寄存器（R32_QSPIx_FCR）

地址：QSPI1=0x40024C0C，QSPI2=0x4002500C，复位值=0x00000000，核心功能：清零SR寄存器中的对应标志位（写1清零）：

  • CTOF（bit4）：清零TOF标志；

  • CSMF（bit3）：清零SMF标志；

  • CTCF（bit1）：清零TCF标志；

  • CTEF（bit0）：清零TEF标志。

3.5 数据长度寄存器（R32_QSPIx_DLR）

地址：QSPI1=0x40024C10，QSPI2=0x40025010，复位值=0x00000000，核心功能：指定间接模式/状态轮询模式下的传输字节数：

  • 实际传输字节数=DL[31:0] + 1；

  • DL[31:0] = 0xFFFFFFFF（全1）：未定义长度，传输至FLASH末尾；

  • 双闪存模式下，DL[0]自动保持1，确保传输偶数个字节；

  • 内存映射模式下无效。

3.6 通信配置寄存器（R32_QSPIx_CCR）

地址：QSPI1=0x40024C14，QSPI2=0x40025014，复位值=0x00000000，核心配置位（命令序列核心配置）：

位字段功能描述实操配置要点
SIOO（bit28）指令仅发送一次，1=仅第一条命令发送指令支持该模式的FLASH可置1，提升传输效率
FMODE[1:0]（bit27~26）功能模式，00=间接写入，01=间接读取，10=状态轮询，11=内存映射根据实际应用场景选择，如读FLASH配置为01b
DMODE[1:0]（bit25~24）数据模式，00=无数据，01=单线，10=双线，11=四线与FLASH的数据传输模式匹配，常用11b（四线）
DCYC[4:0]（bit22~18）空指令周期数，0~31个SCK周期高频时钟下配置，确保FLASH有足够准备时间，如0x0F（15个周期）
ABSIZE[1:0]（bit17~16）交替字节长度，00=8位，01=16位，10=24位，11=32位无需交替字节时，ABMODE=00b，ABSIZE无效
ABMODE[1:0]（bit15~14）交替字节模式，00=无交替字节，01=单线，10=双线，11=四线无需交替字节时配置为00b
ADSIZE[1:0]（bit13~12）地址长度，00=8位，01=16位，10=24位，11=32位根据FLASH容量配置，如16MB FLASH配置为10b（24位地址）
ADMODE[1:0]（bit11~10）地址模式，00=无地址，01=单线，10=双线，11=四线无需地址时配置为00b，常用11b（四线）
IMODE[1:0]（bit9~8）指令模式，00=无指令，01=单线，10=双线，11=四线多数FLASH仅支持单线指令，配置为01b
INSTRUCTION[7:0]（bit7~0）发送给FLASH的8位指令根据FLASH规格配置，如读指令0x03，写使能指令0x06

3.7 其他常用寄存器

  • 地址寄存器（R32_QSPIx_AR）：地址0x40024C18（QSPI1）/0x40025018（QSPI2），存储间接模式/状态轮询模式下的操作地址；

  • 交替字节寄存器（R32_QSPIx_ABR）：地址0x40024C1C（QSPI1）/0x4002501C（QSPI2），存储交替字节数据；

  • 数据寄存器（R32_QSPIx_DR）：地址0x40024C20（QSPI1）/0x40025020（QSPI2），间接模式下用于读写数据，状态轮询模式下存储最新读取的状态数据；

  • 轮询状态屏蔽/匹配寄存器（R32_QSPIx_PSMKR/PSMAR）：地址0x40024C24/28（QSPI1）/0x40025024/28（QSPI2），状态轮询模式下配置屏蔽和匹配条件；

  • 低功耗超时寄存器（R32_QSPIx_LPTR）：地址0x40024C30（QSPI1）/0x40025030（QSPI2），内存映射模式下配置超时时间。

第四章：QSPI实操流程（核心步骤）

以最常用的“间接读取模式（四线）”为例，讲解QSPI的完整实操流程，适配单闪存模式，其他模式可参考此流程调整配置。

4.1 实操准备

  • 硬件连接：将QSPI的SCK、SIO0~SIO3、SCSN引脚与外部SPI FLASH对应引脚连接（如SCK接FLASH的CLK，SIO0接DI，SIO1接DO，SIO2接WP，SIO3接HOLD，SCSN接CS）；

  • 确认FLASH规格：获取FLASH的指令集（如读指令、写使能指令）、通信模式（支持四线）、容量（用于配置FSIZE）。

4.2 初始化步骤

  1. 禁用QSPI：将R32_QSPIx_CR的EN位清0，确保配置前QSPI处于禁用状态；

  2. 等待QSPI空闲：读取R32_QSPIx_SR的BUSY位，确保BUSY=0（IDLEF=1）；

  3. 配置器件参数（R32_QSPIx_DCR）：

    • 配置FSIZE[4:0]：根据FLASH容量配置，如16MB FLASH（2^24），FSIZE=23；

    • 配置CSHT[2:0]：默认000（1个SCK周期）；

    • 配置CKMODE：与FLASH时钟模式匹配，如模式0（CKMODE=0）。

  4. 配置控制寄存器（R32_QSPIx_CR）：

    • 配置PRESCALER[7:0]：根据HCLK频率配置，如HCLK=144MHz，SCK=36MHz，PRESCALER=3（分频系数4）；

    • 配置FTHRES[4:0]：如00010（3字节阈值）；

    • 禁用双闪存模式：DFM=0；

    • 禁用超时计数器：TCEN=0（间接模式无效）；

    • 禁用DMA：DMAEN=0（如需DMA可置1）；

    • 配置中断使能（可选）：如需要传输完成中断，TCIE=1。

  5. 配置通信配置寄存器（R32_QSPIx_CCR）：

    • 指令仅发送一次：SIOO=0（默认，如需可置1）；

    • 功能模式：FMODE=01b（间接读取模式）；

    • 数据模式：DMODE=11b（四线传输）；

    • 空指令周期：DCYC=0x0F（15个SCK周期，根据FLASH规格调整）；

    • 交替字节：ABMODE=00b（无交替字节），ABSIZE无效；

    • 地址长度：ADSIZE=10b（24位地址，适配16MB FLASH）；

    • 地址模式：ADMODE=11b（四线传输）；

    • 指令模式：IMODE=01b（单线传输指令）；

    • 指令：INSTRUCTION=0x03（FLASH读指令，根据FLASH规格调整）。

  6. 配置数据长度（R32_QSPIx_DLR）：如读取4字节数据，DL=0x00000003（4-1=3）；

  7. 使能QSPI：将R32_QSPIx_CR的EN位置1。

4.3 执行间接读取操作

  1. 配置读取地址：将目标地址写入R32_QSPIx_AR寄存器（如0x000000，FLASH起始地址）；

  2. 启动命令：向R32_QSPIx_CR的START位写1，触发读取命令，BUSY位置1；

  3. 等待传输完成：读取R32_QSPIx_SR的TCF位，直到TCF=1（传输完成）；

  4. 读取数据：从R32_QSPIx_DR寄存器读取数据（根据数据长度，多次读取）；

  5. 清零标志位：向R32_QSPIx_FCR的CTCF位写1，清零TCF标志；

  6. 结束操作：如需停止QSPI，将R32_QSPIx_CR的EN位清0。

4.4 其他模式实操要点

（1）间接写入模式实操要点

间接写入模式核心用于向SPI FLASH写入数据，需先执行“写使能”指令，再配置写入相关寄存器，步骤基于前文初始化流程调整，具体如下：

  1. 写使能配置（必做步骤）： 配置R32_QSPIx_CCR寄存器：FMODE=00b（间接写入）、IMODE=01b（单线指令）、INSTRUCTION=0x06（FLASH写使能指令，需匹配FLASH规格）；

  2. 配置R32_QSPIx_DLR：DL=0x00000000（无需传输数据，仅发送指令）；

  3. 启动命令：向R32_QSPIx_CR的START位写1，触发写使能指令，等待TCF=1后清零标志位。

  4. 写入配置调整（基于4.2初始化步骤修改）： R32_QSPIx_CCR调整：FMODE=00b（间接写入）、DMODE=11b（四线传输）、INSTRUCTION=0x02（FLASH页写指令，根据FLASH规格调整）；

  5. R32_QSPIx_DLR：根据写入字节数配置，如写入8字节数据，DL=0x00000007（8-1=7）；

  6. 其余寄存器（DCR、CR）保持初始化配置不变。

  7. 执行写入操作： 写入目标地址：将FLASH写入地址写入R32_QSPIx_AR寄存器；

  8. 启动命令：向R32_QSPIx_CR的START位写1，BUSY位置1；

  9. 写入数据：向R32_QSPIx_DR寄存器依次写入待传输数据，FIFO满时暂停写入，等待FLEVEL降低后继续；

  10. 等待完成：读取R32_QSPIx_SR的TCF位，TCF=1表示写入完成，向R32_QSPIx_FCR的CTCF位写1清零标志；

  11. 写禁止（可选）：写入完成后，可发送写禁止指令（INSTRUCTION=0x04），防止误写入。

注意：FLASH写入前需确保对应扇区已擦除（擦除指令通常为0x20，具体参考FLASH datasheet），未擦除区域无法直接写入。

（2）状态轮询模式实操要点

状态轮询模式主要用于监控FLASH擦除、写入完成状态，以“监控擦除完成”为例，实操步骤如下：

  1. 初始化配置（基于4.2步骤调整）： R32_QSPIx_CCR：FMODE=10b（状态轮询模式）、IMODE=01b（单线指令）、INSTRUCTION=0x05（读取FLASH状态寄存器指令）、DMODE=01b（单线读取状态）、DLR=0x00000000（读取1字节状态）；

  2. R32_QSPIx_CR：配置PMM（如AND模式，PMM=0）、APMS=1（匹配后自动停止轮询）、SMIE=1（使能状态匹配中断）；

  3. 配置匹配条件：假设FLASH擦除完成时，状态寄存器bit0（BUSY位）为0，配置R32_QSPIx_PSMKR=0x00000001（屏蔽其他位，仅监控bit0）、R32_QSPIx_PSMAR=0x00000000（匹配bit0=0）；

  4. 配置轮询间隔：R32_QSPIx_PIR=0x00FF（间隔255个SCK周期，根据需求调整）。

  5. 启动轮询： 发送FLASH擦除指令（如扇区擦除0x20），启动擦除操作；

  6. 向R32_QSPIx_CR的START位写1，启动状态轮询，BUSY位置1；

  7. 轮询过程中，QSPI周期性读取状态寄存器，与PSMAR进行匹配。

  8. 状态匹配处理： 当状态匹配（擦除完成），SMF标志置1，触发中断（SMIE=1时）；

  9. APMS=1时，轮询自动停止，BUSY位清零；

  10. 中断服务函数中，向R32_QSPIx_FCR的CSMF位写1，清零SMF标志，执行后续操作（如通知系统擦除完成）。

（3）内存映射模式实操要点

内存映射模式将FLASH映射到MCU地址空间，无需通过寄存器读写，直接地址访问，实操步骤如下：

  1. 初始化配置（基于4.2步骤调整）： R32_QSPIx_CCR：FMODE=11b（内存映射模式）、IMODE=01b（单线指令）、INSTRUCTION=0x03（读指令）、ADMODE=11b（四线地址）、ADSIZE=10b（24位地址）、DCYC=0x0F（高频时钟下配置）、DMODE=11b（四线数据）；

  2. R32_QSPIx_CR：TCEN=1（使能超时计数器，低功耗优化），配置TIMEOUT[15:0]（如0xFFFF，超时时间根据需求调整）；

  3. R32_QSPIx_DLR：无效，无需配置。

  4. 启动内存映射： 使能QSPI（EN=1），配置目标地址（R32_QSPIx_AR），向R32_QSPIx_CR的START位写1；

  5. BUSY位置1，FLASH正式映射到MCU地址空间（映射地址需参考芯片手册，如QSPI1映射起始地址为0x90000000）。

  6. 地址访问： 直接通过映射地址访问FLASH数据，如*(uint8_t*)0x90000000 即可读取FLASH起始地址的数据；

  7. 仅支持读操作，禁止向映射地址写入数据（会导致传输错误）；

  8. 连续访问时，QSPI会通过FIFO预取数据，提升访问速度。

  9. 停止映射： 向R32_QSPIx_CR的ABORT位写1，中止内存映射，BUSY位清零；

  10. 将R32_QSPIx_CR的EN位清0，禁用QSPI，完成映射停止。

4.5 实操常见问题与解决方法

实操过程中易出现传输失败、数据错误等问题，结合寄存器配置和硬件连接，整理高频问题及解决方案：

  • 问题1：启动命令后，BUSY位始终为1，TCF位不置1原因：寄存器配置错误（如FMODE、DMODE与指令不匹配）、硬件连接错误（SCK、SIO引脚接反）、FLASH未上电或损坏；

  • 解决：核对R32_QSPIx_CCR寄存器配置（重点检查FMODE、IMODE、DMODE），重新检查硬件连接，测量FLASH供电电压，更换FLASH测试。

问题2：读取数据错误，与写入数据不一致原因：SCK时钟频率过高（FLASH不支持）、DCYC空指令周期不足、地址配置错误（ADSIZE与FLASH容量不匹配）；

解决：降低SCK频率（调整PRESCALER分频系数），增加DCYC空指令周期（如调整为0x10），核对FSIZE和ADSIZE配置（如16MB FLASH需FSIZE=23、ADSIZE=10b）。

问题3：状态轮询模式无法触发SMF标志原因：PSMKR（屏蔽寄存器）、PSMAR（匹配寄存器）配置错误，PMM模式选择不当，中断使能未开启；

解决：重新配置PSMKR和PSMAR（确保屏蔽位、匹配位与FLASH状态寄存器对应），根据需求选择PMM模式（AND/OR），确认SMIE位已置1。

问题4：内存映射模式无法访问数据原因：映射地址错误、FMODE未配置为11b、DCYC周期不足、FLASH未正常初始化；

解决：参考芯片手册确认QSPI映射地址，核对R32_QSPIx_CCR的FMODE配置，增加DCYC空指令周期，先通过间接模式验证FLASH可正常访问后，再启动内存映射。

问题5：双闪存模式启动后，吞吐量未翻倍原因：DFM位未置1、两个FLASH数据线连接错误、地址分配未遵循偶奇地址规则；

解决：确认R32_QSPIx_CR的DFM=1，检查SIO0~SIO3（FLASH1）、SIOX0~SIOX3（FLASH2）数据线连接，确保访问字节数为偶数（偶地址对应FLASH1，奇地址对应FLASH2）。

第五章：实操代码示例（基于Keil MDK）

以下提供CH32H417 QSPI间接读取、间接写入、状态轮询的核心代码示例，基于官方标准库编写，适配16MB SPI FLASH（W25Q128），可直接移植使用（需根据实际硬件调整引脚和指令）。

5.1 头文件引用与宏定义

#include "ch32h41x.h"
#include "stdio.h"

// QSPI1寄存器宏定义（QSPI2替换为R32_QSPI2_XXX）
#define QSPI_CR        R32_QSPI1_CR
#define QSPI_DCR       R32_QSPI1_DCR
#define QSPI_SR        R32_QSPI1_SR
#define QSPI_FCR       R32_QSPI1_FCR
#define QSPI_DLR       R32_QSPI1_DLR
#define QSPI_CCR       R32_QSPI1_CCR
#define QSPI_AR        R32_QSPI1_AR
#define QSPI_DR        R32_QSPI1_DR
#define QSPI_PSMKR     R32_QSPI1_PSMKR
#define QSPI_PSMAR     R32_QSPI1_PSMAR
#define QSPI_PIR       R32_QSPI1_PIR

// FLASH参数宏定义（W25Q128）
#define FLASH_READ_CMD  0x03    // 读指令
#define FLASH_WRITE_EN  0x06    // 写使能指令
#define FLASH_WRITE_CMD 0x02    // 页写指令
#define FLASH_ERASE_CMD 0x20    // 扇区擦除指令
#define FLASH_STATUS_CMD 0x05   // 状态读取指令
#define FLASH_SIZE      23      // 16MB FLASH，FSIZE=23
#define FLASH_ADDR_SIZE 0x02    // ADSIZE=10b（24位地址）

5.2 QSPI初始化函数（间接模式，四线）

/**
  * @brief  QSPI初始化（间接模式，适配W25Q128，四线传输）
  * @param  无
  * @retval 无
  */
void QSPI_Init(void)
{
    // 1. 禁用QSPI，等待空闲
    QSPI_CR &= ~(1 << 0);       // EN=0，禁用QSPI
    while((QSPI_SR & (1 << 5)) != 0);  // 等待BUSY=0（空闲）
    
    // 2. 配置器件配置寄存器（DCR）
    QSPI_DCR &= ~(0x1F << 16);  // 清空FSIZE
    QSPI_DCR |= (FLASH_SIZE << 16);  // 配置FLASH容量
    QSPI_DCR &= ~(0x07 << 8);   // 清空CSHT，默认1个周期
    QSPI_DCR &= ~(1 << 0);      // CKMODE=0（模式0）
    
    // 3. 配置控制寄存器（CR）
    QSPI_CR &= ~(0xFF << 0);    // 清空PRESCALER，SCK=HCLK/1
    QSPI_CR |= (0x03 << 0);      // PRESCALER=3，SCK=HCLK/4（HCLK=144MHz，SCK=36MHz）
    QSPI_CR &= ~(0x1F << 8);    // 清空FTHRES，默认0字节阈值
    QSPI_CR |= (0x02 << 8);      // FTHRES=2，阈值3字节
    QSPI_CR &= ~(1 << 6);       // DFM=0，禁用双闪存模式
    QSPI_CR &= ~(1 << 3);       // TCEN=0，禁用超时计数器
    QSPI_CR &= ~(1 << 2);       // DMAEN=0，禁用DMA
    QSPI_CR |= (1 << 17);        // TCIE=1，使能传输完成中断
    
    // 4. 配置通信配置寄存器（CCR）
    QSPI_CCR &= ~(1 << 28);     // SIOO=0，每次命令都发送指令
    QSPI_CCR &= ~(0x03 << 26);  // 清空FMODE
    QSPI_CCR |= (0x01 << 26);    // FMODE=01b（间接读取模式，可后续调整）
    QSPI_CCR &= ~(0x03 << 24);  // 清空DMODE
    QSPI_CCR |= (0x03 << 24);    // DMODE=11b（四线传输）
    QSPI_CCR &= ~(0x1F << 18);  // 清空DCYC
    QSPI_CCR |= (0x0F << 18);    // DCYC=15，15个SCK空周期
    QSPI_CCR &= ~(0x03 << 14);  // ABMODE=00b，无交替字节
    QSPI_CCR &= ~(0x03 << 10);  // 清空ADMODE
    QSPI_CCR |= (0x03 << 10);    // ADMODE=11b（四线地址）
    QSPI_CCR &= ~(0x03 << 8);   // 清空IMODE
    QSPI_CCR |= (0x01 << 8);     // IMODE=01b（单线指令）
    QSPI_CCR &= ~(0xFF << 0);    // 清空指令
    QSPI_CCR |= FLASH_READ_CMD;  // 初始指令为读指令
    
    // 5. 配置数据长度（默认读取4字节）
    QSPI_DLR = 0x00000003;
    
    // 6. 使能QSPI
    QSPI_CR |= (1 << 0);        // EN=1，使能QSPI
    while((QSPI_SR & (1 << 5)) != 0);  // 等待BUSY=0，初始化完成
}

5.3 间接读取函数

/**
  * @brief  QSPI间接读取数据
  * @param  addr：FLASH读取地址（24位）
  * @param  buf：接收数据缓冲区
  * @param  len：读取字节数
  * @retval 0：读取成功，1：读取失败
  */
uint8_t QSPI_Read(uint32_t addr, uint8_t *buf, uint32_t len)
{
    if(len == 0 || buf == NULL) return 1;
    
    // 1. 等待QSPI空闲
    while((QSPI_SR & (1 << 5)) != 0);
    
    // 2. 配置读取参数
    QSPI_CCR &= ~(0x03 << 26);  // FMODE=01b（间接读取）
    QSPI_CCR &= ~(0xFF << 0);    // 清空指令
    QSPI_CCR |= FLASH_READ_CMD;  // 读指令
    QSPI_DLR = len - 1;          // 配置读取长度
    QSPI_AR = addr;              // 配置读取地址
    
    // 3. 启动读取命令
    QSPI_CR |= (1 << 5);        // START=1，启动命令
    while((QSPI_SR & (1 << 5)) != 0);  // 等待BUSY=0
    while((QSPI_SR & (1 << 1)) == 0);  // 等待TCF=1（传输完成）
    
    // 4. 读取数据
    for(uint32_t i=0; i<len; i++)
    {
        buf[i] = QSPI_DR & 0xFF;  // 逐字节读取
    }
    
    // 5. 清零传输完成标志
    QSPI_FCR |= (1 << 1);        // CTCF=1，清零TCF
    
    return 0;
}

5.4 间接写入函数

/**
  * @brief  QSPI间接写入数据（需先擦除对应扇区）
  * @param  addr：FLASH写入地址（24位）
  * @param  buf：待写入数据缓冲区
  * @param  len：写入字节数（不超过FLASH一页大小，通常256字节）
  * @retval 0：写入成功，1：写入失败
  */
uint8_t QSPI_Write(uint32_t addr, uint8_t *buf, uint32_t len)
{
    if(len == 0 || buf == NULL) return 1;
    
    // 1. 写使能
    QSPI_CCR &= ~(0x03 << 26);  // FMODE=00b（间接写入）
    QSPI_CCR &= ~(0xFF << 0);    // 清空指令
    QSPI_CCR |= FLASH_WRITE_EN;  // 写使能指令
    QSPI_DLR = 0x00000000;       // 无数据传输
    QSPI_CR |= (1 << 5);         // 启动命令
    while((QSPI_SR & (1 << 5)) != 0);  // 等待BUSY=0
    QSPI_FCR |= (1 << 1);        // 清零TCF标志
    
    // 2. 配置写入参数
    QSPI_CCR &= ~(0xFF << 0);    // 清空指令
    QSPI_CCR |= FLASH_WRITE_CMD;  // 页写指令
    QSPI_DLR = len - 1;          // 配置写入长度
    QSPI_AR = addr;              // 配置写入地址
    
    // 3. 启动写入命令
    QSPI_CR |= (1 << 5);        // START=1，启动命令
    while((QSPI_SR & (1 << 5)) != 0);  // 等待BUSY=0
    
    // 4. 写入数据
    for(uint32_t i=0; i<len; i++)
    {
        QSPI_DR = buf[i];        // 逐字节写入
        while((QSPI_SR & (0x3F << 8)) == 0x3F << 8);  // 等待FIFO不满
    }
    
    // 5. 等待写入完成
    while((QSPI_SR & (1 << 1)) == 0);  // 等待TCF=1
    QSPI_FCR |= (1 << 1);        // 清零TCF标志
    
    return 0;
}

5.5 状态轮询函数（监控擦除完成）

/**
  * @brief  QSPI状态轮询，监控FLASH擦除完成
  * @param  无
  * @retval 0：擦除完成，1：超时失败
  */
uint8_t QSPI_Poll_Erase_Complete(void)
{
    // 1. 配置状态轮询参数
    QSPI_CCR &= ~(0x03 << 26);  // FMODE=10b（状态轮询模式）
    QSPI_CCR &= ~(0xFF << 0);    // 清空指令
    QSPI_CCR |= FLASH_STATUS_CMD; // 状态读取指令
    QSPI_DLR = 0x00000000;       // 读取1字节状态
    QSPI_PSMKR = 0x00000001;     // 屏蔽其他位，监控bit0（BUSY位）
    QSPI_PSMAR = 0x00000000;     // 匹配bit0=0（擦除完成）
    QSPI_CR &= ~(1 << 23);       // PMM=0（AND模式）
    QSPI_CR |= (1 << 22);        // APMS=1（匹配后自动停止）
    QSPI_CR |= (1 << 18);        // SMIE=1（使能状态匹配中断）
    QSPI_PIR = 0x00FF;           // 轮询间隔255个SCK周期
    
    // 2. 启动轮询
    QSPI_CR |= (1 << 5);        // START=1，启动轮询
    uint32_t timeout = 0xFFFFF;  // 超时计数
    while((QSPI_SR & (1 << 3)) == 0)  // 等待SMF=1（状态匹配）
    {
        timeout--;
        if(timeout == 0) return 1;  // 超时失败
    }
    
    // 3. 清零状态匹配标志
    QSPI_FCR |= (1 &lt;&lt; 3);        // CSMF=1，清零SMF
    return 0;
}

第六章：总结与补充说明

本教程聚焦CH32H417/CH32H416 QSPI接口的实操核心，从基础认知、核心功能、寄存器配置、实操流程到代码示例，覆盖了单/双闪存模式、四种工作模式的关键要点，旨在帮助开发者快速上手。

补充说明如下：

  1. 寄存器地址：QSPI1和QSPI2的寄存器地址不同，代码中需根据实际使用的QSPI外设，替换寄存器前缀（R32_QSPI1_/R32_QSPI2_）；

  2. FLASH适配：不同型号SPI FLASH的指令集、通信模式可能存在差异，实操时需参考对应FLASH的datasheet，调整指令（INSTRUCTION）、DCYC周期等参数；

  3. 中断配置：教程中仅简要提及中断使能，实际应用中需配置NVIC中断优先级，编写中断服务函数（如传输完成、状态匹配中断）；

  4. 低功耗优化：内存映射模式下，可通过TCEN位使能超时计数器，减少FLASH功耗；双闪存模式下，需确保两个FLASH规格一致，避免传输异常；

  5. 官方资源：如需更详细的寄存器说明、时序图，可参考CH32H417/CH32H416官方应用手册和标准库示例代码。

通过本教程的学习，开发者可快速掌握QSPI接口的核心用法，适配不同场景下的SPI FLASH控制需求，后续可根据实际项目需求，灵活调整工作模式和寄存器配置。
